# 中文翻译与验收规范

## 目标读者与文风

- 面向希望真正理解贝叶斯建模的中文读者，保持原书亲切、幽默、略带挑衅的教学语气。
- 优先传达论证关系，不逐词硬译；不得擅自删减限定条件、反例、警告或练习。
- 使用中国大陆简体中文和全角标点。英文缩写与中文之间留半角空格。

## 术语与技术内容

- 以 `translations/zh/glossary.json` 为准；新术语先入表再进入正文。
- `R`、`Stan`、`rethinking`、`quap`、`ulam`、函数名、变量名、文件名与 URL 不翻译。
- 代码逐字保留。代码注释可在不破坏可运行性的前提下译为中文，但需另行审校。
- 数学公式必须从 PDF 渲染图核对，不把 `pdftotext` 的二维公式碎片当作可信来源。
- 模型语句、分布名、参数、上下标和条件关系不得因中文语序改变含义。
- 图、表、代码清单统一写作 `图 1.1`、`表 1.1`、`代码清单 1.1`。

## 页面状态

- 章节级 `in_progress`：章节尚未全部翻译，即使当前已译页面均已验收，也不得把整章标成完成。
- `draft`：已翻译但未逐页核对。
- `reviewed`：已与 PDF 文本层及渲染图核对，术语和语义通过。
- `verified`：静态审计与桌面/移动端视觉审查通过。
- 只有 `verified` 页面计入对外宣称的完成度。

## 每章闭环

1. 用 `scripts/extract_source.py` 生成该章逐页 XML/文本定位材料。
2. 同时查看 `pdftotext -layout` 与 PDF 页面 PNG；遇到公式、图表、浮动元素时以 PNG 为准。
3. 翻译正文，保护代码、公式、引用、专名和链接。
4. 做统计学语义检查、R/Stan 代码检查、中文语言检查和响应式排版检查。
5. 更新 `progress.json`，重新构建并运行 `audit_site.py`。
6. 将可复用的问题与解决办法补充到本文件。

## 发布边界

本项目默认仅供本地学习和审校。未确认权利前，不公开源 PDF、英文抽取物或完整译本；静态站点构建物也不默认配置公网发布。
