# 第二次精译与验收规范

第二次精译不沿用“全书已经完成”的旧状态。它单独记录正文、公式、代码三条审校轨道，并以 PDF 页面为最小单位。`translations/zh/progress.json` 仍表示第一轮完整性；`translations/zh/second-pass.json` 只表示第二轮实际核对过的页面。

## 1. 正文：清楚、自然、准确

- 先判断段落在论证中的作用，再处理句子；中文必须能独立顺畅阅读，同时完整保留原文的条件、转折、因果关系、警告、幽默和语气强弱。
- 不逐词硬译，不用英语语序堆叠长定语。必要时可以拆句或合句，但不得改变论证顺序、指代对象和范围限定。
- 术语以 `translations/zh/glossary.json` 为准。同一概念不随意换译；原书刻意换词或使用双关时，在中文中保留这种差别。
- 人物、机构和地名优先采用通行中文名；首次出现且有辨认价值时，在括号中保留原文，后文使用中文名。参考文献、书名和代码数据中的拼写不改。
- `Bayesian` 在讨论学派、程序或解释时译作“贝叶斯”；作为专名或代码环境的一部分时按原文保留。R、Stan、包名、函数名、变量名、数据集名和 URL 不翻译。
- 每一页同时查看 `pdftotext -layout` 的文字层与 PDF 渲染图。文字层用于检索，渲染图用于确认栏序、脚注、侧栏、图注、连字符和跨页续接。

## 2. 公式：视觉与符号都忠实于原书

- 公式以 PDF 渲染图为唯一视觉依据；不得根据 `pdftotext` 的碎片重建上下标、分式或矩阵。
- 逐项核对变量字母、大小写、粗体、希腊字母、上下标、转置、绝对值、条件竖线、求和/积分范围、括号层级、分布符号与标点。
- 展示公式统一使用 `.equation-block`，并具有稳定、语义化的 `id`。行内公式使用 `.math-inline`；同一公式不得因换行而被拆成语义错误的片段。
- 分式、矩阵和多行模型使用明确的 HTML 结构表达，不用空格模拟二维排版。窄屏允许公式块内部横向滚动，但不缩小到难以辨认。
- 每页即使没有公式，也要在公式轨道中完成“本页无公式”的确认，避免页间遗漏。

## 3. 代码：内容忠实、组件一致

- 可执行部分保持原书的标识符、字符串、数值、函数调用、运算符、参数顺序与输出不变。不得为了中文语序改写代码。
- 叙述性代码注释统一译成简洁中文；数据标签、程序输出、警告、错误信息以及作为教学对象引用的原始文本保持原样。每一处注释都必须与 PDF 对照，不凭上下文补写。
- 所有源码、Stan 片段和输出都使用 `figure.code-listing`。组件必须包含一个直接子级 `figcaption`；源码与输出分别放在普通 `pre` 和 `pre.code-output` 中。
- 编号代码在页面上统一显示为“代码清单 N.N”；非编号片段使用说明性标题，并标出 R、Stan 或 R / Stan。纯警告和续接输出使用诊断样式，不伪装成可执行源码。
- 一页内逐块核对缩进、换行与跨页续接。输出必须跟随产生它的源码；同一清单跨页时明确标注“续”。

## 4. 每章闭环

1. 按章节页码抽取 `-layout` 文本，并把全部页面渲染为 PNG。
2. 逐页精译正文，同时记录疑难术语、勘误与跨页续接。
3. 对该页的每个公式逐符号核对，对每个代码块逐行核对。
4. 更新 `translations/zh/second-pass.json` 的三条页面数组。
5. 运行 `make audit-second-pass`、`make audit` 和 `python3 scripts/audit_coverage.py`。
6. 在桌面宽度与移动端宽度检查受影响页面；公式、代码与宽图不得撑破正文。

只有三条轨道都覆盖该章全部 PDF 页面，才称该章完成第二次精译。全书完成时，`python3 scripts/audit_second_pass.py --require-complete` 必须通过。
