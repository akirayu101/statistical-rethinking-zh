第十一章

上帝给整数加了料

当前译稿对应原书 PDF 第 347–396 页

太空中的寒冷以 Kelvin 命名，约为 3 Kelvin，也就是绝对零度以上约 3 摄氏度。Kelvin 也是苏格兰格拉斯哥附近一条河流的名字；同一条河又把名字给了 William Thomson，也就是 Kelvin 勋爵（1824 年生，1907 年卒）——英国第一位获封贵族头衔的科学家。Thomson 在格拉斯哥的实验室研究热力学，如今太空的寒冷便以一条苏格兰河流为名。

恰如其名，Kelvin 勋爵也研究水。他发明过几台潮汐预测机，本质上是计算潮汐的机械计算机（图 11.1）。所有齿轮与缆索共同组成一组振荡器，能够产生准确的潮汐预测。但观察这样的机器时，你会发现它的大部分结构都在表示内部状态，而不是预测本身。只检查底部某一个齿轮，很难知道潮汐何时到来，因为预测是内部状态组合之后涌现的结果。

广义线性模型（GLM）与这些早期机械计算机非常相像。模型内部活动的部件——参数——相互作用，产生并不显而易见的预测；但我们无法直接读取参数来理解预测。这与前面各章的高斯线性模型很不一样，在那些模型中，单个参数在预测尺度上具有清晰含义。掌握 GLM 需要多投入一点注意力。第一次试图弄清它们如何运转时，它们总是令人困惑。

最常见也最有用的广义线性模型，是计数模型。计数是非负整数——0、1、2，依此类推。它们是一切数学的基础，也是儿童最先学习的内容；但计数模型又复杂得令人陶醉，于是有了本章标题所借用的那句杜撰格言。166关键问题在于：当我们想预测的是计数时，参数的尺度绝不会与结果尺度相同。计数魔像就像潮汐预测机，底层轰鸣运转的机械结构与输出并不相像。只要把潮汐预测机记在心里，你就能掌握这些模型，并负责任地使用它们。

本章将构建两类最常见计数模型的完整实例。我们把一族相关方法称为二项回归：它们都针对二元分类——生/死、接受/拒绝、左/右——建模，并且两类的总数已知。这与第 2 章的弹珠和抛球例子相似，只是现在可以加入预测变量。Poisson 回归是一种对最大值未知的计数建模的 GLM——例如肯尼亚的大象数量、某博士项目收到的申请数量、一期 Psychological Science 中显著性检验的数量。正如第 10 章所述，Poisson 模型是二项模型的特殊情形；本章末尾还会介绍其他几种计数回归。

本章以及后续各章的所有示例，都会使用前面各章介绍的全部工具。正则化先验、信息准则与 MCMC 估计会交织在数据分析示例中。因此，在学习每一种新 GLM 的示例时，你也会练习并加深对以往课程的理解。

11.1 二项回归

回想前几章的抛球模型。那个模型是二项模型，结果是水样本的计数。但它还不是广义线性模型，因为没有预测变量与结果相关联。现在我们的工作，就是把观测计数与那些关联于不同平均计数的变量配对。

二项分布记作：

y ∼ Binomial(n, p)

其中，y 是计数（零或正整数），p 是任意一次“试验”成功的概率，n 是试验次数。上一章已经证明：作为广义线性模型的基础，当每次试验必须产生两种事件之一、且期望值恒定时，二项分布具有最大熵。对于这类变量，不存在熵更高的其他观测前概率假设。在已知取值约束下，它是我们能够使用的最平坦数据先验。

使用二项概率函数的 GLM 有两种常见形式；它们其实是同一个模型，只是数据组织方式不同：

  1. 逻辑斯蒂回归（logistic regression）：数据按单次试验组织，每个结果变量只能取 0 或 1。
  2. 聚合二项回归（aggregated binomial regression）：把协变量取值相同的单次试验聚合在一起。此时结果可以取零，或不超过试验次数 n 的任意正整数。

两种形式使用同一个 logit link 函数（原书第 324 页），所以有时都称为“逻辑斯蒂”回归，因为 logit 函数的逆函数就是 logistic。通过聚合结果变量，可以把逻辑斯蒂形式转换为聚合形式；通过展开结果变量，也可以从聚合形式转换为逻辑斯蒂形式。下面会各自完整演示一个例子。

与其他 GLM 一样，二项回归并不保证产生形状漂亮的多元高斯后验分布，因此二次近似并非总能令人满意。我们会用 quap 处理一些例子，也会用 ulam 做 MCMC 抽样来核对推断。两种方法都做，是为了让你体会：即使原则上不该奏效，二次近似实际上多常能够工作；以及它为什么会在特定情境中失效。这很有用，因为即使你再也不用二次近似，你的频率学派同事仍一直在用，而你也许应当对他们的估计保持一点怀疑。

11.1.1 逻辑斯蒂回归：亲社会黑猩猩

这个例子的数据来自一项旨在评估黑猩猩（Pan troglodytes）亲社会倾向的实验。167实验结构模仿了经济学家与心理学家在人类学生（Homo sapiens studiensis）身上进行的许多常见实验。图 11.2 中，一只焦点黑猩猩坐在长桌一端，面前左右各有一个杠杆。桌面上有四只碟子，其中可能放着它想要的食物。桌子右侧的两只碟子通过机械装置连接右侧杠杆，左侧两只碟子同样连接左侧杠杆。

焦点动物拉动左侧或右侧任一杠杆时，同侧的两只碟子会分别滑向桌子两端，把其中的东西送到相对的两端。在所有实验试次中，焦点动物一侧的两只碟子都装有食物；但桌子另一侧只有一只碟子装有食物。因此，两个杠杆都会把食物送给焦点动物，只有其中一个杠杆还会把食物送到桌子另一端。

实验有两种条件。在伙伴条件下，另一只黑猩猩像图 11.2 那样坐在桌子另一端；在控制条件下，桌子另一端空着。最后，两种平衡处理会交替改变桌子另一端的食物位于左侧还是右侧。这有助于发现每只焦点动物是否偏好使用某一只手。

人类学生参加这类实验时，几乎总会选择连接两份食物的杠杆——也就是亲社会选项——但只在另一名学生坐在桌子对面时如此。驱动本研究的问题是：焦点黑猩猩会不会有相似表现，在另一只动物在场时更常选择亲社会选项？用线性模型的话说，我们想估计条件（另一只动物在场或不在场）与选项（哪一侧是亲社会选项）之间的交互。

只关注比例赔率这类相对效应的风险在于，它不足以告诉我们某个变量是否重要。如果模型中的其他参数让结果本来就极不可能发生，那么即便比例赔率大到 5.0，也不会让结果变得常见。比如某种罕见疾病每百万人只有 1 人患病；再假设阅读本书会让患病赔率增至 5 倍。这大约只意味着每百万人多 4 个病例，总风险变为百万分之五。本书仍可放心阅读。

但也不该忘记相对效应。相对效应是进行因果推断所必需的；当其他基准发生率变化时，它们在条件意义上可能非常重要。想想“相对鲨鱼与绝对企鹅”的寓言。企鹅深受鲨鱼困扰，也非常害怕鲨鱼；人类同样害怕鲨鱼。可我们常听说几乎所有东西都比鲨鱼危险。比如，人死于蜂蜇的概率确实高于死于鲨鱼袭击的概率。这里比较的是绝对风险：一生中死于蜜蜂的风险远高于死于鲨鱼咬伤的风险。

然而，蜜蜂和鲨鱼并不生活在同一个地方，因此这种比较在几乎所有情境中都无关紧要。当你身处水中——企鹅经常如此——真正想知道的是死于鲨鱼袭击的相对风险。以“人在海里”为条件，鲨鱼远比蜜蜂危险。这就是为什么从企鹅视角看，死于鲨鱼咬伤的相对风险是有价值的信息。对人类也一样，在少数进入海洋的时候，我们想知道的正是相对鲨鱼风险。企鹅身处鲨鱼出没水域的基准发生率要高得多，因此相对风险对于鲜美无比的企鹅高度相关。

再思考：Simpson 悖论不是悖论

这个经验例子在统计教学中非常著名，常用来说明一种称为 Simpson 悖论的现象。169和大多数悖论一样，这里没有违反逻辑，违反的只是直觉。既然不同人的直觉不同，Simpson 悖论对不同人也会有不同含义。本例违背的糟糕直觉是：整个总体中的正关联，也应在每个院系内部成立。总体而言，这批数据中的女性确实更难被研究生院录取；但原因是女性申请了那些无论男女都最难录取的院系。

也许更像悖论的是，这种现象可以在一个样本内无限重复。每当向模型加入另一个预测变量时，结果与某个预测变量之间的任何关联都可能归零或反转。这种反转可能揭示真实的因果影响，也可能只是混杂造成的，正如第 6 章的祖父母例子。我们唯一能做的，是始终对模型保持怀疑，并努力设想它可能怎样欺骗我们。以因果方式思考这类情境通常会有所帮助。170

11.2 Poisson 回归

当结果是从零到某个已知上界的计数时，二项 GLM 很合适。如果可以把数据类比成抛球模型，就应使用二项 GLM。但很多时候，上界并不知道，计数也从未接近任何上限。例如我们捕鱼回来带着 17 条鱼，理论最大值是多少？不管是多少，它都不在数据里。该怎样对鱼的数量建模？

事实证明，只要换个角度看，二项模型仍然适用。当二项分布中事件概率 p 很小、试验次数 N 很大时，它会呈现一种特殊形状。二项分布的期望值就是 Np，方差则为 Np(1 − p)。当 N 很大而 p 很小时，二者近似相等。

再假设一番帝国戏剧之后，你又拥有了另一座修道院。你不知道里面有多少修士辛勤工作，但顾问告诉你，它平均每天产出 2 份手稿。只凭这条信息，就可以推断每天完成手稿数量的完整分布。

用 Poisson 分布构建模型时，模型形式甚至比二项模型或高斯模型更简单。这是因为 Poisson 分布只有一个描述其形状的参数，于是数据概率可以写成：

yi ∼ Poisson(λ)

参数 λ 是结果 y 的期望值，同时也是计数 y 的期望方差。

还需要一个 link 函数。Poisson 模型的惯用 link 函数，是上一章（原书第 326 页）介绍的 log link。为了嵌入线性模型，我们写成：

yi ∼ Poisson(λi)
log(λi) = α + β(xi − x̄)

log link 确保 λi 始终为正，而计数结果的期望值必须为正。但正如上一章所说，它也意味着预测变量与期望值之间是指数关系。指数关系增长得非常快，几乎没有哪种自然现象能长期保持指数增长。因此，使用 log link 时始终要检查的一件事，是它在预测变量的整个取值范围内是否合理。log 尺度上的先验也会以令人意外的方式缩放，所以先验预测模拟在这里仍然很有帮助。

11.2.2 负二项（gamma-Poisson）模型

Poisson 模型通常会留下大量无法解释的变异。这些额外变异大概来自随案例而变的未观测影响，使真实的 λ 彼此不同。忽略这种变异，也就是率异质性，会像在二项模型中一样造成混杂。因此，Poisson GLM 一种非常常见的扩展，是把 Poisson 分布换成负二项分布。它其实是伪装起来的 Poisson 分布，也因此有时称为 gamma-Poisson 分布：它是不同 Poisson 分布的混合。下一章会处理混合模型。

11.2.3 示例：暴露量与 offset

参数 λ 是 Poisson 模型的期望值，但也常被理解为一个率。两种解释都正确；意识到这一点，就能为各案例 i 暴露量不同的情况构建 Poisson 模型。例如，邻近修道院每周汇总完成的手稿，而你的修道院每天汇总。如果同时得到两套记录，面对不同时间跨度、不同暴露量下聚合的计数，该怎样在同一个模型中分析？

方法如下。隐含地，λ 等于单位时间或距离 τ 内的期望事件数 μ。这意味着 λ = μ/τ，从而可以重新定义 link：

yi ∼ Poisson(λi)
log λi = logμiτi = α + βxi

比值的对数等于对数之差，因此也可以写成：

log λi = log μi − log τi = α + βxi

这些 τ 值就是“暴露量”。如果不同观测 i 的暴露量不同，那么第 i 行的期望值为：

log μi = log τi + α + βxi

当 τi = 1 时，log τi = 0，就回到了起点。暴露量随案例变化时，τi 会正确缩放每个案例的期望事件数。因此，不同暴露量的案例可以用下面的模型处理：

yi ∼ Poisson(μi)
log μi = log τi + α + βxi

其中 τ 是数据中的一列。这就像把暴露量的对数作为预测变量加入，却不给它新增参数。也可以在 log τi 前放一个参数，用来表达事件率并非随时间恒定的假设。

最后一个 Poisson 例子考虑观测间暴露量不同的情形。观测时长、采样面积或采样强度变化时，观测到的计数自然也会变化。Poisson 分布假设事件率在时间（或空间）上恒定，所以处理起来很容易：只需把暴露量的对数加入线性模型。这个加入项通常称为 offset。

11.3 多项与分类模型

二项分布适用于只有两种可能事件、而我们对它们计数的情况。一般而言，可能发生的不止两件事。回想第 2 章的弹珠袋，里面原本只有蓝、白两色弹珠；若再加入红色，每次抽取就可能属于三类，累计计数也横跨三类，最终得到蓝、白、红弹珠各自的数量。

当可能出现两种以上无序事件，且每类事件概率在试验间恒定时，最大熵分布是多项分布。第 10 章把石子投入桶中介绍最大熵时，我们已经隐式遇到过它。二项分布其实只是多项分布的特例，因此公式也像把二项分布外推到三种或更多事件。若有 K 类事件，其概率为 p1, …, pK，那么在总共 n 次试验中，观察到各类事件数量 y1, …, yK 的概率为：

Pr(y1, …, yK | n, p1, …, pK) = n!∏i yi! K∏i=1 piyi

分式顶部的 n! 只是表示：有多少种不同排列会产生相同的计数 y1, …, yK。这就是上一章著名的多重度。

基于多项分布的模型也可称为分类回归，通常指像逻辑斯蒂回归一样，每个事件单独占一行的组织方式。在机器学习中，这类模型有时称为最大熵分类器。从多项 likelihood 构建 GLM 很复杂：事件类型增多时，建模选择也随之增多，而且构造 likelihood 还有两条不同路线。第一条直接基于多项 likelihood，使用 logit link 的推广；下面会给出一个例子，称为显式方法。第二条则出人意料地把多项 likelihood 转换成一系列 Poisson likelihood，稍后再介绍。

这里惯用且自然的 link 是多项 logit，也称 softmax 函数。它接收一个分数向量——K 类事件各有一个分数——并计算特定事件类型 k 的概率：

Pr(k | s1, …, sK) = exp(sk)K∑i=1 exp(si)

rethinking 包以 softmax 函数提供这个 link。与它结合后，这类 GLM 可称为多项逻辑斯蒂回归。

最大的问题，是怎样处理多个线性模型。在二项 GLM 中，可以任选两类事件之一，为其 log odds 构建一个线性模型，另一类会自动得到处理。但多项（或分类）GLM 对 K 类事件需要 K − 1 个线性模型。选择一个结果值作为“枢轴”，其余类别都相对于它建模。每个线性模型都可以使用任意预测变量和参数；它们不必相同，而且往往确有理由不同。只有两类事件时，这些选择都不会出现，因为只有一个线性模型；所以二项 GLM 容易得多。

存在两种基本情形：（1）预测变量对不同结果值取不同数值；（2）每个结果值拥有不同参数。第一种适用于每类事件都有自己的定量特征、而我们希望估计这些特征与该事件出现概率之间关联的情况。第二种适用于我们关心产生事件的某个实体之特征，而无论最终产生哪一类事件。下面分别考察两种情形，并各用一个有经验动机的例子加以说明。两种情形可以混进同一个模型，不过先看各自的纯粹例子，更容易把握区别。

本节将用纯 Stan 代码构建模型。我们也可以使用 quap 或 ulam，但直接使用 Stan 能更清楚地展示：管理多个并行线性模型需要怎样的数据结构。以后要按自己的用途修改模型——包括加入变化效应和其他小玩意——也会更容易。172

11.4 删失与生存

有时，对离散、可计数事件建模的正确方法，并不是建模计数本身，而是建模事件之间的时间。例如，假设我们关心动物收容所中猫被领养的速率。一只猫只能被领养一次——至少要等它再次被送养，才可能又被领养。它等待领养的时长为领养速率提供了信息，模型还可以告诉我们速率怎样随品种或毛色而变化。也许你不关心猫的领养（你这个怪物），但你大概会关心疾病发作与康复的速率、死亡时间、首次繁殖年龄，以及许多结构相似的变量。

处理这类数据的模型称为生存模型。生存模型研究的是可计数事物，但我们希望预测的结果是持续时间。持续时间是相对于某个参照点的连续位移，因此都是正实数。

距离同样是正实数。这两类测量都是位移，建模方式也非常相似。位移最简单的分布是指数分布：当我们只知道数值的平均位移时，它就是最大熵分布。因此，如果目标只是估计事件的平均发生率，它是最保守的选择。Gamma 分布也很常用；当均值与平均量级（对数）固定时，Gamma 是最大熵分布。下面的“过度思考”框给出这两种分布的一些生成模型。还有许多其他选择，都允许事件率随时间变化。为简单起见，我们从指数分布开始。

保持结果分布简单的一个理由是，生存模型真正棘手的地方并不是为持续时间指定什么概率分布，而是怎样处理删失。当关注的事件没有在观测窗口内发生时，就出现删失。最简单的情形是，观测在事件发生前结束。例如，收容所里仍有猫在等待领养，其中许多最终会被领养。另一种删失，是其他事件发生后让关注的事件不再可能发生；例如，一只猫可能在等待领养时寿终正寝。

不能直接丢弃被删失的个体。想象 100 只猫同时开始等待领养，一个月后其中一半已被领养。领养率是多少？不能只用已被领养的猫计算，还必须考虑尚未被领养的猫。那些目前还没被领养、但最终会被领养的猫，等待时间显然比已经被领养的猫更长。因此，只看已领养猫得到的平均速率会向上偏——它被“以是否领养为条件”所混杂。

纳入删失观测并不困难，但需要本书此前未见过的一类模型。关键在于，对结果使用同一个分布假设，既能告诉我们以事件结束的任一已观测持续时间的概率，也能告诉我们等待同样长时间却仍未看到事件的概率。这确实是一种奇怪的生物。不妨先从生成模型——一次模拟——出发，为统计模型建立直觉，然后再分析一份真实的猫领养样本。

11.5 总结

本章介绍了一些最常见的广义线性模型，也就是用于对计数建模的模型。分析前绝不要把计数换算成比例，因为这样会破坏样本量信息。这些模型的一个根本难点是：参数与所描述的结果变量处在不同尺度上——二项模型通常使用对数赔率，Poisson 模型通常使用对数率。因此，计算模型的隐含预测比以往更加重要。

    yi ∼ Binomial(ni, pi)
    log pi1 − pi = α + βPPi + βVVi + βAAi

    βV ∼ Normal(0, 0.5)
    βA ∼ Normal(0, 0.5)
