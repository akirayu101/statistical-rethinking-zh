第七章

尤利西斯的罗盘

当前译稿对应原书 PDF 第 211–228 页

米科瓦伊·哥白尼（Mikołaj Kopernik，亦称 Nicolaus Copernicus，1473–1543）：波兰天文学家、教会法学家，以及亵渎神明者。他以太阳系日心模型闻名，主张用日心模型取代地心模型，因为前者更“和谐”。几十年后，这个立场最终导致伽利略与教会著名的不和，以及那场审判。

这个故事后来成了科学战胜意识形态与迷信的寓言。但撇开意识形态，哥白尼的论证如今看来并不充分，问题有两个：他的模型既不特别和谐，也不比地心模型更准确。哥白尼模型十分复杂，实际上与托勒密模型一样堆满本轮（图 7.1）。哥白尼把太阳移到中心，但仍坚持用完美圆形轨道，所以仍然需要本轮；“和谐”并不很能形容它的外观。与托勒密模型一样，哥白尼模型实质上也是傅里叶级数——一种近似周期函数的方法。

这又引出第二个问题：日心模型与地心模型给出完全相同的预测。无论假设地球静止还是运动，都可以构造等价近似，所以只凭准确性没有理由偏爱日心模型。

不过，哥白尼并非只诉诸含糊的和谐。他还以所需原因更少来论证模型优越：“我们由此追随自然；自然不做无用或多余之事，常常宁愿让一个原因产生许多效应。”94 确实，要给出与地心模型相同的预测，日心模型需要的圆和本轮更少；从这个意义上说，它更简单。

学者常偏爱更简单的理论。有时这种偏好很模糊，只是一种审美；有时则退回实用主义，因为简单理论的模型更容易处理。科学家经常援引一个宽松原则——奥卡姆剃刀：应优先选择假设更少的模型。对哥白尼与托勒密而言，剃刀给出明确建议。它不能保证哥白尼正确（毕竟他并不完全正确），但既然日心与地心模型预测相同，至少能明确化解两难。

然而，更一般地使用剃刀很困难，因为通常要在准确性与简单性都不同的模型之间选择。二者怎样权衡？剃刀没有指引。本章介绍一些最常用的权衡工具。所有工具通常都包含某种简单性概念，所以常被比作奥卡姆剃刀；但它们同样关心提高预测准确性，并会明确权衡准确性与简单性，因此并不等同于剃刀。

因此，与其想象奥卡姆剃刀，不如想象尤利西斯的罗盘。尤利西斯是荷马《奥德赛》的英雄。航行途中，他必须穿过一条狭窄海峡：一侧是从悬崖扑下、吞食水手的多头怪兽斯库拉，另一侧是把船只与人拖入水墓的海怪卡律布狄斯。离任何一边太近都是灾难。对科学模型而言，可以把两只怪物看成两类根本统计错误：

  1. 过拟合的多头怪兽：从数据中学得太多，导致预测很差；
  2. 欠拟合的漩涡：从数据中学得太少，同样导致预测很差。

还有第三只怪物，就是前几章认识的混杂。本章将看到：受混杂的模型事实上可能比正确测量因果关系的模型预测得更好。因此设计任何统计模型时，都必须决定目标是理解原因，还是仅仅预测。二者不是同一个目标，需要不同模型。但即使要准确测量因果影响，也仍需处理过拟合；无论目标是什么，过拟合和欠拟合总在暗处潜伏。

我们的任务是小心在这些怪物之间航行。常见方法有两大家族。第一类使用正则化先验，告诉模型不要因为数据而过度兴奋；非贝叶斯方法把同一装置称为“惩罚似然”。第二类使用某种评分装置，例如信息准则或交叉验证，模拟预测任务并估计预测准确性。这两类方法在自然科学和社会科学中都很常用，而且可以——也许应该——结合使用，值得理解。

为了介绍信息准则，本章还必须介绍信息论。若是第一次接触，它大概会显得奇怪，但理解一点信息论是必要的。一旦开始使用信息准则——本章将介绍 AIC、DIC、WAIC 和 PSIS-LOO——就会发现，实现它们远比理解它们容易，这是它们的诅咒。因此本章大部分内容都用来对抗这道诅咒，重点解释概念基础，应用随后再来。

还应提前说明：这些材料很难。若中途感到困惑，那很正常。困惑只是大脑在正确校准主题难度；随着时间推移，困惑会被理解取代，你会逐渐熟悉过拟合、正则化和信息准则在常见情境中的行为。

再思考：数星星

实践科学家最常见的模型选择方法，是寻找一个所有系数都具有统计显著性的模型。统计学家有时称这种做法为“数星星”（stargazing），因为它表现为扫描估计值后面的星号（**）。作者的一位同事曾借 A. C. Clarke 的小说与电影，把它叫作“太空漫游”：星星最多的模型便是最好的。

但这种模型并不最好。无论怎样看待零假设显著性检验，用它在结构不同的模型之间做选择都是错误的；p 值不是为了帮助我们在欠拟合和过拟合之间航行而设计。开始使用 AIC 等度量后会看到，改善预测的变量未必统计显著，统计显著的变量也可能对预测毫无用处。传统 5% 阈值纯属约定，所以不应期待它优化任何东西。

再思考：AIC 是贝叶斯的吗？

AIC 通常不被视为贝叶斯工具，既有历史原因，也有统计原因。历史上，AIC 最初推导时没有引用贝叶斯概率；统计上，它使用 MAP 估计而非完整后验，而且要求平坦先验，所以看起来并不特别贝叶斯。另一个名为贝叶斯信息准则（BIC）的模型比较指标，也强化了这种印象；然而 BIC 同样要求平坦先验和 MAP 估计，而且其实并不是“信息准则”。

尽管如此，AIC 在贝叶斯概率下有清晰、务实的解释，Akaike 等人也长期为它提出其他贝叶斯论证。95 本书后面还会看到，当 AIC 的假设成立时，WAIC 这类更明显的贝叶斯信息准则给出的结果几乎与 AIC 完全相同。从这个角度，完全可以把 AIC 看作 WAIC 一类贝叶斯准则的特殊极限，即使它最初并非如此推导。这也体现了统计程序的常见特征：同一程序可以从多种、甚至哲学上不相容的视角推导和论证。

7.1 参数的问题

前几章看到，向模型加入变量与参数可以揭示隐藏效应、改善估计；也看到加入变量可能有害，尤其在缺乏可信因果模型时。碰撞点是真实存在的。但有时我们并不关心因果推断，只想做出好预测。以前一章的祖父母—父母—儿童例子为例，把所有变量都加入模型，确实会得到良好预测；至于我们是否理解发生了什么，并不重要。那么，把一切都加入模型可以吗？

答案是“不可以”。不断加入变量有两个相互关联的问题。第一个是：加入参数、使模型更复杂，几乎总会改善模型对数据的拟合。96 这里的“拟合”是指模型对拟合它所用数据进行回溯预测的好坏程度。这样的度量有很多，每一种都有自己的怪脾气。在线性高斯模型中，最常用的是 R2，它常被描述为“解释的方差”，定义为：

R2 =var(结果) − var(残差)var(结果)= 1 −var(残差)var(结果)

R2 容易计算，因此很受欢迎。但它和其他样本内拟合度量一样，会随着预测变量的增加而上升；即使新加入的变量只是与结果毫无关系的随机数，也是如此。因此，只看模型对现有数据的拟合，不能用来选择模型。

第二，复杂模型虽然对现有数据拟合得更好，预测新数据时却往往更差。参数很多的模型通常比简单模型更容易过拟合：它会对用于拟合的确切样本极其敏感；未来数据只要和过去不完全相同，就可能产生巨大错误。参数过少的简单模型则更容易欠拟合：无论未来数据多么像过去数据，它都会系统性地高估或低估。因此，不能总偏爱简单模型，也不能总偏爱复杂模型。

下面用一个简单例子同时考察这两个问题。

7.1.1 更多参数（几乎）总会改善拟合

当模型从样本中学得太多时，就发生了过拟合。每个样本都同时具有规则特征和不规则特征。规则特征是我们想学习的目标，因为它们能很好地推广，或者能回答感兴趣的问题；在我们选定的目标下，它们是有用的。不规则特征则是数据中不能推广的部分，因此可能误导我们。

遗憾的是，过拟合会自动发生。在本书目前见过的这类统计模型中，加入额外参数总会改善模型对样本的拟合。本书后面从第 13 章开始，会遇到另一类模型：加入参数不一定改善样本内拟合，却很可能提高预测准确性。

脑大小与身体大小之间最简单的模型是线性模型，我们先从它开始。写出模型之前，先重新缩放变量。前几章已说明，重新缩放预测变量和结果变量，往往有助于模型拟合，也有助于指定和理解先验。这里把体重标准化为均值 0、标准差 1；脑容量则除以观测到的最大值，让最大值变为 1。为什么不把脑容量也标准化？因为我们想保留零这个参照点——完全没有脑。脑不可能是负的。我想是这样。

第一个线性模型的数学表达为：

bi ∼ Normal(μi, σ)
μi = α + βmi
α ∼ Normal(0.5, 1)
β ∼ Normal(0, 10)
σ ∼ Log-Normal(0, 1)

这个模型只是说，物种 i 的平均脑容量 bi 是其体重 mi 的线性函数。再看这些先验意味着什么。α 的先验以数据中重新缩放后的平均脑容量为中心；也就是说，具有平均体重的普通物种，其脑容量的 89% 相容区间大约从 −1 到 2。这个范围荒谬地宽，还包含不可能的负值。β 的先验非常平坦，以 0 为中心，允许荒谬巨大的正关联和负关联。尤其当模型越来越复杂时，这些先验会允许荒谬的推断。这正是本例教训的一部分，所以先继续拟合：

再思考：OLS 与贝叶斯反本质主义

也可以用非贝叶斯的普通最小二乘法（ordinary least squares，OLS），为这些脑大小模型取得后验分布。例如，用 R 的简单函数 lm 就能得到 m7.1 的后验分布；不过得不到 σ 的后验。原文此处误写为 m6.1，代码与上下文都表明应为 m7.1。

只要先验足够模糊，使观测值到回归线的平方偏差之和最小，就等价于寻找后验均值。事实上，卡尔·弗里德里希·高斯最初正是在贝叶斯框架中推导 OLS 的。98 当时几乎所有概率论都是贝叶斯式的，尽管“贝叶斯”这个词要到 20 世纪才广泛使用。

多数情况下，非贝叶斯程序都有近似的贝叶斯解释；这一点对两个方向都很有力量。把非贝叶斯程序改写成贝叶斯解释，会把假设重新表述为信息，因而非常有助于理解程序为何有效。反过来，贝叶斯模型也可以由效率更高、但近似的非贝叶斯程序来体现。贝叶斯推断的含义是近似后验分布，并不规定必须怎样完成这种近似。

这个例子并不是要赞美 R2，而是要埋葬它。但下葬之前仍得先计算它，好在这很容易。先为每个观测计算后验预测分布——前几章已经用 sim 做过。再从每个预测中减去对应观测，得到残差。接着需要残差和结果变量各自的方差。这里要的是实际的经验方差，而不是 R 函数 var 返回的方差；后者是频率学派估计量，分母不适合这里的目的。因此用老办法计算方差：到均值的平方偏差的平均值。rethinking 包为此提供了 var2 函数。

原则上，贝叶斯方法要求对后验的每一个样本都这样计算。不过传统上，R2 只在平均预测处计算，所以这里也照此办理。

而需要预测脑大小的物种恰好也有 7 个。因此，这个模型实际上给每个观测到的脑大小分配了一个独有参数，用来逐一重现它们。这是普遍现象：只要采用参数足够多的模型族，就能精确拟合数据。但这样的模型会对尚未观测的案例做出相当荒谬的预测。

再思考：把模型拟合看作压缩

还可以换个角度理解上面的荒谬模型：模型拟合可被视为一种数据压缩。参数汇总数据之间的关系，把数据压缩成更简单的形式，不过会丢失样本的一些信息，也就是“有损”压缩。随后又可用这些参数生成新数据，相当于把数据解压。

当模型像 m7.6 一样，每个数据点都有一个对应参数时，实际上没有发生压缩；模型只是用参数把原始数据换一种形式编码。因此，我们无法从这样的模型中学到有关数据的任何东西。要了解数据，必须使用更简单的模型，实现一定程度、但又不过度的压缩。这种模型选择观点通常称为最小描述长度（Minimum Description Length，MDL）。99

7.1.2 参数太少也有害

过拟合的多项式模型能极好地拟合数据，却以这种样本内准确性为代价，做出毫无意义的样本外预测。相比之下，欠拟合产生的模型在样本内和样本外都不准确：它们学得太少，未能恢复样本中的规则特征。

理解欠拟合模型的另一个办法，是注意它对样本不敏感。从样本中删除任意一个点，得到的回归线几乎一样。相反，最复杂的模型 m7.6 对样本极其敏感；删除任意一点，预测均值都会大幅改道。

图 7.4 清楚展示了这一点。两幅图都逐次删除数据的一行，并重新推导后验分布。左图每条线都是一次多项式 m7.1，分别拟合由删除一行产生的七套可能数据；右图的曲线则是不同的四次多项式 m7.4。直线几乎不变，曲线却四处乱飞。这是欠拟合与过拟合模型之间的一般差异：它们对用于拟合的确切样本构成，敏感程度不同。

再思考：偏差与方差

欠拟合与过拟合的二分，经常被描述为偏差—方差权衡。100 两者虽不完全相同，偏差—方差权衡处理的却是同一个问题：“偏差”与欠拟合有关，“方差”与过拟合有关。

不过这些术语容易混淆，因为即使在统计学内部，它们在不同语境中的用法也各不相同。“偏差”听起来还像坏事，尽管提高偏差往往能带来更好的预测。因此本书更喜欢欠拟合/过拟合，但应当预期会看到类似例子被称为偏差/方差。

7.2 熵与准确性

那么，怎样在过拟合的多头怪兽与欠拟合的漩涡之间航行？无论最终使用正则化、信息准则，还是两者并用，第一件事都是选定模型表现的判据。你希望模型在哪件事上做得好？我们把这个判据称为目标；本节会说明信息论怎样提供一个通用而有用的目标。

不过，通往样本外离差的道路颇为曲折，前方分三步。第一，需要建立一个度量尺度，衡量离完美准确还有多远。这要借助一点信息论，因为它能为两个概率分布之间的距离提供自然尺度。第二，需要把离差确立为“离完美准确的相对距离”的近似。最后，还要说明我们真正关心的只有样本外离差。掌握以离差度量模型表现后，后续小节会展示正则化先验与信息准则怎样帮助改善并估计模型的样本外离差。

这部分材料很复杂，不必第一次就全部理解。

7.2.1 解雇天气预报员

准确性取决于怎样定义目标，不存在普遍最优的目标。定义目标时，需要操心两个主要维度：

  1. 成本—收益分析。预测错误要付出多大代价？预测正确又能获得多大收益？多数科学家从未正式提出这些问题，但应用科学家必须经常回答它们。
  2. 语境中的准确性。有些预测任务天生比另一些容易。因此即使忽略成本与收益，仍需一种判断“准确性”的方法，把模型究竟还有多少潜在改进空间考虑进去。

用一个例子探索这两个维度会更清楚。假设某座城市有一位天气预报员，每天对全年是下雨还是晴天给出不确定预测。101 预测以降雨概率表示。现任预报员对连续十天给出了如下预测，每个概率下方是实际天气：

7.2.1.2 度量准确性

即使忽略依据预报做实际决策的成本与收益，采用哪种“准确性”度量仍有歧义。“命中率”并没有任何特殊之处。真正应该聚焦的问题是：知道生成数据的真实模型时，哪一种“准确性”定义会达到最大？我们当然不可能做得比真实模型更好。

考虑计算准确预测整段十日序列的概率。也就是先计算每天预测正确的概率，再把所有概率相乘，得到正确预测观测序列的联合概率。这与联合似然完全相同；此前一直用它配合贝叶斯定理拟合模型。正确模型会使这种准确性定义达到最大。

从这个角度看，新人的表现更糟。现任预报员的概率为 13 × 0.47 ≈ 0.005；新人则是 03 × 17 = 0。新人正确预测整段序列的概率为零，因为他的预测从不预期下雨。所以尽管新人“平均来看正确”的概率——命中率——很高，其联合正确概率却糟透了。

现任：13 × 0.47 ≈ 0.005
新人：03 × 17 = 0

联合概率正是需要的度量。为什么？因为它作为似然出现在贝叶斯定理中。它是唯一能正确计数每个事件——雨晴序列——可能发生的相对方式数的度量。还可以比较最大化平均概率与最大化联合概率的后果。真实的数据生成模型不会拥有最高命中率；天气预报例子已经表明，给下雨分配零概率会提高命中率，却显然是错的。相反，真实模型会拥有最高联合概率。

统计文献有时把这种准确性度量称为对数评分规则，因为通常计算并报告联合概率的对数。如果某项分析使用别的东西，那么它要么是对数评分规则的特例，要么就是错误。

再思考：什么是真模型？

所有模型都是假的，因此很难定义“真实”概率。这里的“真实”是什么意思？它表示在我们的无知状态下正确的概率。无知状态由模型描述；概率存在于模型中，而不在世界中。如果掌握了生成预报所需的一切相关信息，雨或晴将是确定的，“真实”概率只会是 0 或 1。缺少某些相关信息时——所有建模都是如此——结果在小世界中不确定，尽管在大世界中仍完全确定。正因为无知，我们才可能拥有介于 0 与 1 之间的“真实”概率。

例如像第 2 章那样投掷地球仪。接住之前，结果并不确定；以假定模型为条件，观测到水域有一个“真实”概率。但如果掌握地球仪投掷的足够信息——初始条件、角动量向量等等——结果就能被确定地预知。没有两次投掷会完全相同，所以观测到水域的“真实”概率必须对未知差异取平均，以描述水域相对陆地的可信程度。这个模型提出的问题有一个正确答案——70% 水域。正是我们对投掷物理过程的无知，使它能成为估计地表水域比例的方法。

7.2.2 信息与不确定性

现在想用数据的对数概率为竞争模型的准确性评分。下一个问题是怎样度量离完美预测的距离。完美预测只需报告每天降雨的真实概率。因此，只要任一预报员给出的预测不同于目标，就可以度量预测离目标有多远。但该采用什么距离？答案并不显然，却存在唯一且最优的答案。

找到答案，需要先理解准确性指标应当完成什么任务。它应该认识到，有些目标天生比另一些容易命中。例如把天气预报延伸到冬季，现在一天可能下雨、晴天或下雪。犯错的方式从两种变成三种。任何合理的“离目标距离”都必须反映这一点，因为增加一种事件后，目标更难命中了。

这好比把二维射箭靶变成还必须在正确时间命中的三维目标。最好与最差射手之间可能出现的距离变大了，因为多出一种脱靶方式；同理，也多出一种让优秀射手给人留下深刻印象的方式。目标与射击之间的潜在距离越大，潜在改进空间以及优秀射手惊艳我们的能力也越大。

20 世纪 40 年代末，人们给出了度量模型准确性与目标距离的方案。102 信息论最初用于电报等消息通信问题，如今已广泛影响基础科学与应用科学，并与贝叶斯推断有深刻联系。不过，与许多成功领域一样，信息论也催生了大量牵强应用。103

核心洞见是问：得知一个结果后，我们的不确定性减少了多少？ 再看天气预报。预报提前发布，当时天气不确定；实际日期到来后，天气便不再不确定。因此，不确定性的减少自然衡量了我们学到多少、从观测结果中获得多少“信息”。若能精确定义“不确定性”，就能获得一条基准，衡量预测有多难，以及存在多大改进空间。在这里，经度量的不确定性下降就是信息的定义。

信息：得知一个结果时，不确定性的减少。

使用这个定义，需要一种有原则的方法量化概率分布内在的不确定性。再次假设某天只有两种可能天气：晴或雨。每种事件以某个概率发生，而这些概率之和为 1。我们想要一个函数，把晴雨概率转换成不确定性的度量。

度量不确定性的方法很多。最常见的途径先规定这种度量应具有的性质。下面是三条直观判据：

  1. 连续性。不确定性度量应当连续。否则，任一概率——例如降雨概率——任意微小的变化，都会造成不确定性的巨大跳变。
  2. 可能事件越多，不确定性越高。例如有两座城市需要天气预报。第一座城市一年中一半日子下雨，其余日子晴天；第二座城市下雨、晴天和下冰雹各占三分之一。第二座城市多一种需要预测的事件，所以我们希望它的不确定性度量更大。
  3. 可加性。如果先分别度量雨或晴（两个事件）以及热或冷（另两个事件）的不确定性，那么四种组合——雨/热、雨/冷、晴/热、晴/冷——的不确定性，应等于两项独立不确定性之和。

只有一个函数满足这三条判据。它通常称为信息熵，定义出奇地简单。若有 n 种可能事件，事件 i 的概率为 pi，并把概率列表记作 p，那么所求的唯一不确定性度量为：

H(p) = − E log(pi) = − ∑i=1n pi log(pi)(7.1)

概率分布所含的不确定性，是一个事件的平均对数概率的负值。

这里的“事件”可以是一种天气——例如雨或晴——也可以是一种鸟类，甚至 DNA 序列中的某个特定核苷酸。

不必深入推导 H 的细节，但应该指出，这个函数没有任何部分是任意规定的；每一部分都来自上面三项要求。尽管如此，我们接受 H(p) 作为有用的不确定性度量，并非只因导出它的前提，而是因为实践证明它极其有用且富有成果。

一个例子有助于消除 H(p) 的神秘感。计算天气的信息熵时，假设雨和晴的真实概率分别为 p1 = 0.3 与 p2 = 0.7，那么：

H(p) = −(p1 log(p1) + p2 log(p2)) ≈ 0.61

用 R 计算：

若改住在阿布扎比，雨和晴的概率也许更接近 p1 = 0.01 与 p2 = 0.99，此时熵约为 0.06。为什么不确定性降低了？因为阿布扎比几乎不下雨，所以与降雨概率为 30% 的地方相比，任意一天的不确定性小得多。信息熵正是这样度量事件分布内在的不确定性。

类似地，若给分布增加一种事件——把预报延伸到冬季，同时预测降雪——由于预测问题增加一个维度，熵往往上升。例如晴、雨、雪的概率分别为 0.7、0.15、0.15，熵约为 0.82。

不过，这些熵值本身对我们意义不大。接下来要用它们构造准确性度量。

过度思考：再谈熵

上面说信息熵是平均对数概率，但定义中还带有一个 −1。把平均对数概率乘以 −1，只是让熵 H 从零向上增加，而不是从零向下减小；这是惯例，不影响功能。上面使用自然对数（底为 e），换底只会重新缩放，不影响推断。以 2 为底的二进制对数也同样常见。只要比较的所有熵都使用相同底数，就没有问题。

计算 H 的唯一技巧，是处理不可避免的 pi = 0。log(0) = −∞，不能直接使用；但洛必达法则告诉我们 limpi→0 pi log(pi) = 0。因此计算 H 时，把 0 log(0) 视为 0。换句话说，永不发生的事件会退出计算；事件既然永不发生，也就没有理由把它留在模型里。

再思考：最大化不确定性的益处

信息论有许多应用，其中尤其重要的是最大熵（maximum entropy，也称 maxent）。最大熵是一族方法，用来寻找与某种知识状态最一致的概率分布。换句话说：给定已知信息，哪一种分布最不令人意外？其中一个答案是在以先验知识为约束的条件下，使信息熵最大。104

这样做实际上会得到后验分布。因此，贝叶斯更新就是熵最大化。最大熵将在第 10 章占据重要位置，帮助我们构建广义线性模型（GLM）。

7.2.3 从熵到准确性

有一种量化不确定性的方法很好，H 已经提供了它。现在可以精确表述目标有多难命中。但怎样用信息熵说明模型离目标有多远？关键在于散度：

散度：用一个分布的概率描述另一个分布时，引入的额外不确定性。

它常称为 Kullback–Leibler 散度，或简称 KL 散度，以为此目的引入它的两位学者命名。105

例如，假设事件的真实分布是 p1 = 0.3、p2 = 0.7，而我们认为事件以 q1 = 0.25、q2 = 0.75 的概率发生。用 q = {q1, q2} 近似 p = {p1, p2}，会引入多少额外不确定性？正式答案以 H 为基础，公式同样简单：

DKL(p, q) = ∑i pi[log(pi) − log(qi)] = ∑i pi log(pi / qi)

通俗地说，散度是目标 p 与模型 q 的对数概率之差的平均值。它就是两个熵之差：目标分布 p 的熵，与使用 q 预测 p 所产生的交叉熵之差。当 p = q 时，我们知道事件的实际概率。在这种情况下：

DKL(p, q) = DKL(p, p) = ∑i pi[log(pi) − log(pi)] = 0

用一个概率分布表示自身，不会引入任何额外不确定性。这多少令人安心。

更重要的是，q 与 p 越不相同，散度 DKL 就越大。图 7.5 给出一个例子。假设真实目标分布为 p = {0.3, 0.7}，近似分布 q 可以从 {0.01, 0.99} 变化到 {0.99, 0.01}。横轴显示第一个概率 q1，纵轴显示散度 DKL(p, q)。只有恰好在 q = p、即 q1 = 0.3 时，散度为零；其他地方都会增大。

散度现在能帮助我们比较 p 的不同近似。近似函数 q 越准确，DKL(p, q) 越小。因此若有一对候选分布，使散度最小的候选最接近目标。预测模型会为事件——观测——指定概率，所以可以用散度比较模型的准确性。

过度思考：交叉熵与散度

散度的推导比想象中容易。关键是认识到：用概率分布 q 预测来自另一分布 p 的事件时，会定义交叉熵：

H(p, q) = −∑i pi log(qi)

事件按照 p 发生，却按照 q 被预期，因此熵会膨胀；膨胀多少取决于 p 与 q 有多不同。散度定义为使用 q 所引入的额外熵，所以只是事件实际熵 H(p) 与交叉熵 H(p, q) 之差：

DKL(p, q) = H(p, q) − H(p)
= −∑i pi log(qi) − (−∑i pi log(pi))
= −∑i pi[log(qi) − log(pi)]

因此，散度确实以熵为单位，度量 q 离目标 p 有多远。注意谁是目标非常重要：一般而言，H(p, q) 不等于 H(q, p)。下面的“再思考”侧栏会进一步说明。
