缺失数据与其他机会

贝叶斯推断的一大优点，是不必非得聪明才行。例如，有一道经典概率谜题叫作Bertrand 盒子悖论。210 我偏爱的版本用煎饼来讲。假设我煎了三张饼：第一张两面都焦了（BB），第二张只焦了一面（BU），第三张两面都没焦（UU）。现在我随机拿一张给你，盘中朝上的一面恰好焦了。另一面也焦了的概率是多少？

若依靠直觉，这道题很难。大多数人会回答“二分之一”，但这错得相当厉害。绝非故作谦虚，我的直觉也没好到哪里去；不过，我已经学会用冷酷、坚决、无情地应用条件概率来解决这类问题。既然可以无情，就不必聪明。

那就无情起来。应用条件概率，是用我们已经知道的事来修正对想知道之事的认识。换句话说：

Pr(想知道 ∣ 已经知道)

本例中，我们知道朝上的一面焦了，想知道朝下的一面是否也焦了。条件概率的定义告诉我们：

Pr(朝下焦 ∣ 朝上焦) = Pr(朝上焦，朝下焦)Pr(朝上焦)

这只是条件概率的定义，换成了煎饼问题的标签。我们想知道朝下的一面是否焦了，掌握的信息则是朝上的一面焦了。以这条信息为条件，就会据此更新信息状态。定义表明，所求概率就是“两面都焦”这张饼的概率，除以看见焦面朝上的概率。由于饼是随机抽取的，两面都焦那张饼的概率为 1/3；而焦面朝上的概率，必须对所有可能拿到焦面朝上的方式取平均：

Pr(朝上焦) = Pr(BB)(1) + Pr(BU)(0.5) + Pr(UU)(0)
= (1/3) + (1/3)(1/2) = 0.5

合在一起：

Pr(朝下焦 ∣ 朝上焦) = 1/31/2 = 23

如果还不太相信，可以用一个快速模拟确认：

三分之二。

既然已经看过正确答案，若想在最后补上一点直觉，诀窍是数煎饼的面，而不是数煎饼本身。确实，有两张饼至少一面焦了，而其中只有一张两面都焦；但真正重要的是饼面，不是饼。以朝上一面焦了为条件，朝下的一面可能来自三个饼面，其中两个焦了，所以概率是 3 个中的 2 个。

概率论在数学上并不难，无非是计数；难的是解释和应用。这样做往往显得需要几分机巧，作者也有动机用巧妙方法解题来炫技。但只要无情地应用条件概率，我们不需要那种聪明。贝叶斯方法真正的诀窍，正是对所有地方——数据和参数——都应用条件概率。好处是，一旦定义了信息状态，也就是我们的假设，就能让概率规则完成其余工作；真正完成的工作，是揭示假设所蕴含的后果。一直以来练习的模型拟合，也是同一种不聪明的方法：定义模型、放入数据，随后条件概率完成其余工作，依据证据揭示假设的含义。

本章会介绍这种“先假设、再演绎”策略的两个常见应用。第一，把测量误差纳入模型；第二，通过贝叶斯插补估计缺失数据。两者都会给出完整的入门示例。

在这两种应用中，设计模型时都无需凭直觉猜测测量误差的后果，也无需猜测缺失值意味着什么。只要陈述自己掌握的误差信息，或带缺失值变量的信息，逻辑便会完成其余工作。好吧，是计算机完成其余工作；但它只是用复杂算法执行贝叶斯更新，谈不上聪明。它揭示的含义却既反直觉，又很有价值。

再思考：生成式思考，贝叶斯推断

贝叶斯模型是生成式的：既能估计参数，也能模拟观测。这样做的一个好处，是我们可以从数据如何产生出发，认真构造统计模型。这不仅要考虑研究过程本身，也要考虑抽样与测量过程；随后让贝叶斯更新揭示这些假设的含义。有时，结论恰恰是无法从现有数据推断生成过程。贝叶斯是个诚实的伙伴，不怕伤害你的感情。

接下来该怎么办？注意 Dobs 是 D 的后代。用前者代替后者不一定会引入混杂。实际上，大多数回归很可能都在使用 Dobs 这类代理，因为大多数变量都带有一定测量误差。不过，即使它不一定打开非因果路径，使用代理也可能引入系统偏差，扭曲估计。本例中，各州测量误差的大小还会以某种方式和目标变量关联。

若能使用 D 而不是 Dobs，当然会更好，但我们没有 D。不过，可以尝试重建它，同时尊重不确定性、避免虚假自信。在这些数据中，报告的标准误 Divorce.SE 是根据产生误差 eD 的过程计算出来的。怎样在统计模型中使用这条信息？这就像把模拟倒过来。

若要模拟测量误差，会给每次观测指定一个分布并从中抽样。例如，假设某次测量的真实值为 10 米，以标准差 2 米的高斯误差测量，那么任意实现的测量值 y 都服从：

y ∼ Normal(10, 2)

测量误差越小，概率就越集中在 10 附近；只要存在误差，10 上下的一系列测量值就都有可能出现。这就是所谓“普通数据是分布的特例”。关键洞见是：如果不知道真实值 y（本例中为 10），就用参数表示它，再让贝叶斯更新完成其余工作。

下面为每个离婚率定义误差分布。每个观测值 Dobs,i 都对应一个参数 Dtrue,i：

Dobs,i ∼ Normal(Dtrue,i, DSE,i)

这只是把测量值 Dobs,i 定义为一个指定的高斯分布，以未知参数 Dtrue,i 为中心。因此，上式为每个州的观测离婚率指定了一个概率，同时使用已知测量误差。若从已知真实离婚率模拟观测离婚率，会写成：

信息很多，但仍然一步一步来。目标是把离婚率 D 建模成结婚年龄 A 与结婚率 M 的线性函数。把测量误差突出显示后，模型为：

15.2 缺失数据

对于测量误差，关键洞见是：任何不确定的数据项，都能用反映其不确定性的分布来替代。但有时数据就是缺失了——根本没有任何测量。乍看之下，这似乎无计可施：连带误差的测量都不存在时，还能做什么？

处理缺失值最常见的方式，是删除所有含任意缺失值的案例；这称为完整案例分析，也是大多数统计软件默认而无声的行为。另一种常见做法，是用某个假定值替换缺失值，例如变量均值或零这样的参考值。这两种处理都不安全。完整案例分析最好也只是低效，因为它丢弃数据；而且是否产生偏差取决于因果细节。用固定值替换缺失值永远没有正当理由——我们并不知道那些值；一旦把它们固定，模型就会以为自己确定无疑地知道它们。

那还能怎么做？我们可以从因果角度思考缺失性，并用模型插补缺失值。生成模型会告诉你，产生缺失值的过程是否也会妨碍识别因果效应；有时会，有时不会。幸好，可以把缺失性加入 DAG，再用已经学过的同一套准则判断它是否造成混杂。生成模型还会提供关于尚未看见之数值的信息。213 这些信息可用来对不确定性取平均，充分利用非缺失值，而不必丢掉任何东西。

再思考：缺失数据是有意义的数据

一个变量的值未被观测到，这件事实本身仍是一项观测。它也是数据，只不过取了一个非常特殊的值；这个值的含义取决于情境。以个人收入问卷为例：若有人拒绝填写收入，拒答可能与低收入或高收入有关，因此，试图预测缺失值的模型可能很有启发。在生态学中，没有观察到某个物种也是一种微妙的观测：它可能表示该物种不在那里，也可能表示它在那里，只是你没有看到。为处理这种双重性，已有完整一类模型称为占域模型。214 缺失值总是由某个过程产生，思考这个过程有时能解决大问题。

15.2.2 插补灵长类

处理缺失数据往往涉及对缺失值进行插补。插补既能避免有偏估计，也让我们能够使用全部已观测（即未缺失）的数据。插补的关键思想是：任何生成模型都必然包含关于未观测变量的信息。某些数据会缺失，但模型保持不变。因此，理论上插补缺失数据很容易；实践中则一如既往地可能遇到挑战。

贝叶斯插补的统计诀窍，是为含有缺失值的变量建立模型。每个缺失值都分配一个独有参数；观测值会提供关于数值分布的信息，而该分布会成为缺失值的先验，随后由完整模型更新。

这样，每个缺失值都会得到一个后验分布。从概念上说，这和测量误差的情形相似：不知道的量用参数表示，再以已知信息为条件，通过贝叶斯更新推断其余部分。

本例中，含缺失值的变量是新皮层百分比，仍称作 B，代表“brain”：

B = [0.55, B2, B3, B4, 0.65, 0.65, …, 0.76, 0.75]

每个缺失值所在的索引 i，都有一个参数 Bi 为它形成后验分布。最简单的模型直接从 B 自身的正态分布插补 B。模型如下：

Ki ∼ Normal(μi, σ)[结果 K 的分布]
μi = α + βBBi + βM log Mi[线性模型]
Bi ∼ Normal(ν, σB)[观测/缺失 B 的分布]
α ∼ Normal(0, 0.5)
βB ∼ Normal(0, 0.5)
βM ∼ Normal(0, 0.5)
σ ∼ Exponential(1)
ν ∼ Normal(0.5, 1)
σB ∼ Exponential(1)

该模型忽略了 B 与 M 通过 U 相关。为保持简单，先从它开始。起初，Bi ∼ Normal(ν, σB) 的解释有些别扭：当 Bi 已观测时，这一行就是似然，和普通线性回归一样；模型会学习与数据一致的 ν 与 σB 分布。但当 Bi 缺失、因而成为参数时，同一行被解释成先验。由于参数 ν 与 σB 也由估计得到，这个先验会像前几章的变化效应一样从数据中学习。

这个模型有一个问题：它假设每个 B 值的不确定性都服从标准化高斯分布。然而这些值是比例，必然介于 0 与 1 之间，所以模型还能改进；章末练习会展示具体方法。不过要记住，为变量指定高斯分布，并不等于断言它的经验频数分布一定呈钟形；这里只表示我们用均值和方差描述它。在方差给定时，高斯分布是最平坦的无界分布（第 10 章），因而是相当保守的选择。但正如第 7 章早已说明，若有理由怀疑分布具有厚尾，就绝对不要使用高斯分布。

插补模型有多种实现方式。每种方式都稍显别扭，因为必须尊重缺失值的位置，这意味着大量索引管理。这里采用与上述讨论紧密对应的方法：把观测值与参数合并成一个向量，在回归中把它当作“数据”。为方便起见，ulam 可以自动完成这种合并。本节末的“过度思考”框会给出原始 Stan 代码的完整实现。

再思考：多重插补

缺失数据插补有一段混乱的历史。插补形式很多，其中大多是缺乏坚实概率论基础的临时办法：热卡插补、冷卡插补、均值替代、随机插补等等。这些程序如今没有一个被视为体面的方法。常见的非贝叶斯程序是多重插补。216 多重插补是在调查无应答的背景下发展出来的，实际上有贝叶斯理由支撑。但它发明于桌面计算机还无法实际执行贝叶斯插补的年代，所以试图近似“随机缺失”缺失性模型的完整贝叶斯解。若你不愿删除不完整案例，那么采用多重插补也应当让你感到放心。

该程序从缺失值的近似后验分布中进行多次抽取，分别用这些抽取值开展分析，再以近似完整贝叶斯插补的方式合并各次分析。多重插补比完整贝叶斯插补更受限制，所以现在我们直接使用真正的完整方案。但许多非贝叶斯分析仍在使用多重插补。记住，频率主义统计并不是关于如何产生估计的理论，而只是一套关于如何评价估计的理论。

15.3 类别误差与离散缺失

上面的示例都聚焦于处理方便的连续变量。测量误差一节中的变量是连续的，缺失数据一节中的新皮层百分比也是连续的。变量连续时，可以直接为每个未知值分配一个参数——无论它是带测量误差，还是完全缺失——再让 Markov 链承担困难工作。

但若变量是离散的——例如 0/1 或 1、2、3、4——Markov 链就需要额外辅导。离散的未观测变量要求离散参数，而离散参数有两个问题。第一，离散变量不会产生可供 Hamiltonian Monte Carlo 平滑滑行的曲面；HMC 根本无法处理离散变量。第二，其他估计方法也会遇到离散参数空间的问题，因为离散跳跃很难校准，链往往会长时间卡住。

但这不表示我们束手无策。几乎在每种情形下，都根本不需要抽样离散参数；可以使用一种专家称作“加权平均”的特殊技术，把离散参数从模型中移除。对其他参数完成抽样后，再用其样本计算被移除离散参数的后验分布。因此不会放弃任何信息。无论使用哪种引擎，移除离散参数实际上都会提高 Markov 链效率，所以即便不用 HMC，通常也值得这样做。参数不离散时，这种技术有时仍然有用，因为移除连续参数也会加快链。

这一切听来好得不像真的。它全都是真的，但实现方法一点也不显然。本节会用最简单的示例教你怎样做。无论情境如何，关键思想都是：变量是否被观测（数据）或未被观测（参数），它的信息都由生成模型定义。本节会用到一点数学，但不会超过中学所学。掌握一般方法后，就能应用到非二元离散变量，包括计数变量和类别变量。

+ Pr(Ci = 0) Pr(Di = 1) Pr(Ni | Ci = 0, Di = 1)
+ Pr(Ci = 0) Pr(Di = 0) Pr(Ni | Ci = 0, Di = 0)

若只有猫为 NA，而且已知狗存在（Di = 1），只需对可能性（1）和（3）取平均：

Pr(Ni) = Pr(Ci = 1) Pr(Ni | Ci = 1, Di = 1) + Pr(Ci = 0) Pr(Ni | Ci = 0, Di = 1)

若只有狗为 NA，而且已知猫不存在（Ci = 0），则对可能性（3）和（4）取平均：

Pr(Ni) = Pr(Di = 1) Pr(Ni | Ci = 0, Di = 1) + Pr(Di = 0) Pr(Ni | Ci = 0, Di = 0)

原则上，这套方法算法明确而且简单。实践中，代码却会很复杂，因为必须照顾所有缺失组合，并给每一种组合分配不同的平均似然。

下一章遇到状态空间模型时，还会再见到这项一般技术。状态空间模型可能包含大量离散（或连续）未观测变量。通常不会在代码里逐一写出每种可能，而是用一种算法遍历所有可能性，计算必要的平均似然。例如在隐 Markov 模型中，会用一种叫作前向算法的算法完成取平均；Stan 用户手册给出了一个示例。

15.3.2 离散误差

上面的示例处理缺失数据。但若数据不是缺失，而是带有测量误差，做法也非常相似。例如，仍用前面的例子，假设为每栋房子分配一个猫存在的概率，称为 ki。确信有猫时，ki = 1；确信没有猫时，ki = 0；觉得正反各半时，ki = 0.5。这些 ki 值会取代前一模型中的参数 k，成为对不确定性取平均时使用的权重。

15.4 总结

本章快速介绍了测量误差模型与缺失数据模型的设计和实现。测量误差与缺失数据都有原因；把这些原因纳入生成模型，有助于判断误差与缺失性怎样影响推断，也有助于设计统计程序。本章突出贯穿全书的一般原则：有效的统计建模，既要求仔细思考数据如何生成，也要求精细处理数值算法。任何一项都无法独自托起推断。
