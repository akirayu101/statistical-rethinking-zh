第十四章

协方差历险

当前译稿对应原书 PDF 第 465–512 页

回想上一章导言中的咖啡机器人（原书第 413 页）。这个机器人被设定为在咖啡馆之间移动、点咖啡，并记录等待时间。上一章的重点是：在咖啡馆之间汇聚信息，能让机器人学得更高效；变化截距正是实现这种汇聚的一种机制。

现在假设机器人还记录一天中的时段——上午或下午。平均而言，上午等待时间往往长于下午，因为咖啡馆上午更忙。然而，咖啡馆不仅有不同的平均等待时间，上午与下午之间的差异也各不相同。在传统回归中，这些上午—下午等待时间差就是斜率，因为它们表示时段指示变量（或虚拟变量，原书第 157 页）改变取值时，期望会怎样变化。线性模型可以写成：

μi = αCAFÉ[i] + βCAFÉ[i]Ai

其中，Ai 是表示下午的 0/1 指示变量，βCAFÉ[i] 是每家咖啡馆下午与上午预期等待时间差的参数。

机器人汇聚截距信息时，能更高效地学习上式中的截距 αCAFÉ[i]；同样地，若也汇聚斜率信息，它就能更高效地学习斜率。实现方式相同：在估计每条斜率的同时，估计斜率的总体分布。模型（机器人）从数据中学习先验，而赋给截距与斜率的分布则让两者都能汇聚。

这就是一般变化效应策略的实质：凡是一批以可交换值作索引的参数，都可以——而且大概都应该——汇聚。可交换只表示索引值没有真实顺序，因为它们不过是任意标签。截距并不特殊；斜率也能随数据中的单元而变化，而在斜率之间汇聚信息同样能更充分地利用数据。因此，应当把咖啡机器人设定为同时建模截距总体与斜率总体，让它对两者都使用汇聚，从数据中榨出更多信息。

不过，还有一个事实能榨出更多信息：咖啡馆的截距和斜率会协变。为什么？在一家热门咖啡馆，员工上午非常忙，平均等待时间很长（图 14.1）；但同一家店下午会清闲许多，因此上午与下午的等待时间差很大。这样的热门咖啡馆截距高，斜率也远离零。相反，不太热门的咖啡馆上午等待较短，下午改善也不大。在包含热门与冷门店的整个咖啡馆总体中，截距与斜率会协变。

这种协变本身就是机器人能够利用的信息。若能跨参数类型——截距和斜率——汇聚信息，机器人上午学到的东西就能改进它对下午的学习，反之亦然。比如，机器人上午抵达一家新咖啡馆，观察到咖啡等了很久；即使它还没在同一家店的下午点过咖啡，也能更新对下午等待时间的预期，因为在咖啡馆总体中，上午久等与下午等待更短相关。

本章将展示怎样真正做到这一点：把变化斜率与上一章的变化截距结合起来。这样的汇聚会改进对不同单元如何响应预测变量、或如何受预测变量影响的估计；它还能跨参数类型借用信息，从而改进截距估计。从本质上说，变化斜率模型是庞大的交互作用机器：数据中的每个单元都可以对任何处理、暴露或事件作出独特响应，同时仍通过汇聚改进估计。当斜率变异很大时，平均斜率就不那么值得关注。有时，斜率变异的模式还会提示遗漏变量，解释为何某些单元反应更强或更弱。本章会看到一个例子。

本章稍后还会把支持复杂变化效应的机制推广到更微妙的模型，包括用高斯过程处理连续类别。普通变化效应只适用于离散、无序的类别，例如个人、国家或池塘；在这些情形中，每个类别与其他类别的差异被同等看待。然而，汇聚也能用于年龄或位置等类别，因为某些年龄、某些位置会比另一些更相似。我们将学习怎样建模这类连续类别之间的协变，并把策略推广到看似无关的模型，例如系统发育回归和网络回归。最后还会回到因果推断，利用对协方差的新能力超越第 6 章的工具，引入工具变量。工具变量是一种无需关闭后门路径也能推断因果的方法，不过它们在设计和估计上都非常棘手。

14.1 从构造出发理解变化斜率

机器人应当怎样跨截距和斜率汇聚信息？答案是建模截距与斜率的联合总体，也就是建模它们的协方差。在传统多层模型中，实现这一点的装置，是为全部变化效应——截距和斜率——指定一个联合多元高斯分布。因此，不再分别为截距和斜率指定两个相互独立的高斯分布，而是把截距放在第一维、斜率放在第二维，为两者共同指定一个二维高斯分布。

从第 4 章开始使用二次近似表示后验分布时，你就已经在处理多元高斯分布。拟合模型的方差—协方差矩阵 vcov，描述每个参数的后验概率怎样与其他参数的后验概率关联。现在，我们要用同一类分布描述不同种类变化效应内部的变异，以及它们之间的协变：变化截距有变异，变化斜率有变异，截距与斜率还会协变。

为了看清它怎样工作，以及怎样指定和解释变化斜率，我们来模拟导言中的咖啡机器人。与先前的模拟练习一样，这既会再次强调 Bayesian 统计模型的生成性质，也会帮助你理解怎样开展自己的前瞻性功效分析。

再思考：为什么是高斯分布？

截距与斜率的多元分布并非一定要是高斯分布，但这里同时存在实践与认识论上的理由。实践上，容易处理的多元分布并不多，常见的只有多元高斯与多元 Student-t 分布。认识论上，如果我们只想陈述这些截距和斜率的均值、方差与协方差，那么最大熵分布就是多元高斯。不过，高斯分布的薄尾仍可能带来风险。

过度思考：方差、协方差与相关

在典型的统计用法中，我们用三个参数定义协方差：（1）第一个变量的标准差，例如 σα；（2）第二个变量的标准差，例如 σβ；（3）两个变量之间的相关，例如 ραβ。为什么协方差等于 σασβραβ？

变量 x 与 y 之间的协方差通常定义为：

cov(x, y) = E(xy) − E(x)E(y)
var(x) = cov(x, x) = E(x2) − E(x)2

也可以说，“协方差等于乘积的平均值减去平均值的乘积”。方差只是它的特殊情形，即变量与自身的协方差。若只考虑期望为零的随机变量——这没有损失，因为随时可以重新居中——那么就只剩 cov(x, y) = E(xy) 与 var(x) = E(x2)。

相关只是重新缩放后的协方差，使最小值为 −1、最大值为 1。用最大可能的协方差去除，就能把协方差标准化；这个最大值是 √{var(x)var(y)}，也就是两个标准差的乘积。

下面说明它为何确实是 cov(x, y) = E(xy) 能取得的最大值。当第二个变量 y 只是 x 的重新缩放副本时，协方差最大。比如令 yi = p xi，其中 p 是 0.5 或 1.5 之类的比例。于是 y = px 只是被拉伸的 x，而 cov(x, y) = E(px2) = pE(x2)。同时，var(x) = E(x2)，var(y) = E(p2x2) = p2E(x2)。还觉得有趣吗？马上就结束。两者方差乘积的平方根正好是 pE(x2) = cov(x, y)；这就是协方差能达到的最大值。

因此，若想得到标准化的关联度量——相关——就用协方差除以这个最大值：

ρxy = cov(x, y)√{var(x)var(y)}

把这个方程对 cov(x, y) 求解，就得到：

cov(x, y) = √{var(x)var(y)}ρxy = σxσyρxy

呼。以上只是为了说明，应用统计中把协方差写成 cov(x, y) = σxσyρxy，既有充分依据，也很方便。

在本例中，一切都很平衡：每家咖啡馆恰好观测 10 次，时段也始终平衡，每家店上午、下午各 5 次。但一般而言，数据不必平衡。与上一章的蝌蚪示例一样，不平衡实际上会让变化效应分析更占优势，因为部分汇聚能在最需要的地方利用总体信息。

再思考：模拟与模型设定偏误

在这项练习中，我们从一个生成过程模拟数据，再用结构完全正确地反映该过程的模型分析数据。但现实世界里从没有这么幸运；我们总被迫用存在设定偏误的模型分析数据，也就是模型不同于真实的数据生成过程。不过，模拟可以探索模型设定偏误：先从一个过程模拟数据，再观察若干都不与真实生成过程完全一致的模型表现如何。还要始终记住，Bayesian 推断并不依赖 likelihood 等数据生成假设为真。非 Bayesian 方法的推断可能依赖抽样分布，Bayesian 模型却并非如此；在 Bayesian 模型中，likelihood 是数据的先验，而对参数的推断可能对其细节惊人地不敏感。

过度思考：多层模型的非中心化参数化

当链的效率低下时，把链运行得足够久，往往仍能从后验分布获得可靠样本。正文中的 m14.2 就是这样。但这既低效又不可靠：链仍可能以难以察觉的细微方式带有偏差。更好的做法是按前一节的解释重新参数化。195 遇到协方差矩阵时，具体该怎样做？

模型 m14.3 用一个名为 Cholesky 分解的技巧，把协方差矩阵从先验中“偷运”出来。模型顶部与中心化版本 m14.2 相同，变化在于增加了构造自适应先验的几行：

# 自适应先验——非中心化
transpars> matrix[actor,4]:alpha <-
        compose_noncentered( sigma_actor , L_Rho_actor , z_actor ),
transpars> matrix[block_id,4]:beta <-
        compose_noncentered( sigma_block , L_Rho_block , z_block ),
matrix[4,actor]:z_actor ~ normal( 0 , 1 ),
matrix[4,block_id]:z_block ~ normal( 0 , 1 ),

前两行以 transpars> 开头，定义变化效应矩阵 alpha 与 beta。每个矩阵中，一行对应一个行动者或区组，一列对应一种效应。为了方便，compose_noncentered 把标准差向量、Cholesky 相关因子与 z 分数混合起来，生成在线性模型中具有正确尺度的参数矩阵。这样一来，上面第三、第四行的 z 分数矩阵只需服从 normal(0,1)。要把模型改为非中心化，另一处变化是把普通相关矩阵替换成所谓的 Cholesky 因子，确切类型为 cholesky_factor_corr。

那么 compose_noncentered 到底在做什么？神秘的 Cholesky 因子又是什么？Cholesky 分解 L 能把相关矩阵 R 这类方形对称矩阵表示成 R = LL⊺。一个奇妙事实是：用 L 乘以一组不相关样本（z 分数）组成的矩阵，便会得到一组相关样本（变化效应）组成的矩阵。正是这个技巧让我们能把协方差矩阵拿出先验：先抽取不相关 z 分数矩阵，再乘以 Cholesky 因子和标准差，就能得到尺度与相关都正确的变化效应。它看似魔法，其实只是代数。

    matrix[7,4] alpha;
    matrix[6,4] beta;

这些计算把标准差向量 sigma_actor、sigma_block 与 Cholesky 相关因子 L_Rho_actor、L_Rho_block 合并起来。diag_pre_multiply 所做的，就是把 sigma 向量变成对角矩阵后再相乘，从而得到正确协方差矩阵的 Cholesky 因子。最后，再把这个 Cholesky 协方差因子与 z 分数矩阵作矩阵乘法。为方便起见，结果还要转置——即每行末尾的 '——这样便可写成 alpha[actor,effect]，而非 alpha[effect,actor]；不过这一步并非真正必需。

随后在 model 块中，alpha 与 beta 矩阵已经可以直接作为参数使用，所以线性模型部分看起来与中心化版本相同：

model{
    vector[504] p;
    L_Rho_block ~ lkj_corr_cholesky( 2 );
    sigma_block ~ exponential( 1 );
    L_Rho_actor ~ lkj_corr_cholesky( 2 );
    sigma_actor ~ exponential( 1 );
    g ~ normal( 0 , 1 );
    to_vector( z_block ) ~ normal( 0 , 1 );
    to_vector( z_actor ) ~ normal( 0 , 1 );
    for ( i in 1:504 ) {
        p[i] = g[tid[i]] + alpha[actor[i], tid[i]] + beta[block_id[i], tid[i]];
        p[i] = inv_logit(p[i]);
    }
    L ~ binomial( 1 , p );
}

从上到下看：先声明向量 p，用来保存每个观测的线性模型计算；接着用 Cholesky 相关因子与标准差向量定义先验。z 分数矩阵通过 to_vector 获得先验，因为 normal(0,1) 作用于向量而非矩阵。z 分数仍以矩阵形式存储；to_vector 只是为了强制矩阵中的每个单元格都服从相同的 normal(0,1) 先验。最后，线性模型使用转换参数块中的 alpha 与 beta 矩阵完成计算，再照常定义数据的概率。

最后一部分是生成量块，可以在这里计算每个样本的函数。本例用它把 Cholesky 因子转换成普通相关矩阵，以便按相关矩阵解释；同时还计算 WAIC 或 PSIS 所需的对数概率：

generated quantities{
    vector[504] log_lik;
    vector[504] p;
    matrix[4,4] Rho_actor;
    matrix[4,4] Rho_block;
    Rho_block = multiply_lower_tri_self_transpose(L_Rho_block);
    Rho_actor = multiply_lower_tri_self_transpose(L_Rho_actor);
    for ( i in 1:504 ) {
        p[i] = g[tid[i]] + alpha[actor[i], tid[i]] + beta[block_id[i], tid[i]];
        p[i] = inv_logit(p[i]);
    }
    for ( i in 1:504 ) log_lik[i] = binomial_lpmf( L[i] | 1 , p[i] );
}

multiply_lower_tri_self_transpose 只是把 Cholesky 因子 L 转换成对应矩阵 R = LL⊺ 时，执行所需矩阵代数的一种紧凑高效写法。

这些非中心化形式显然有代价：看起来混乱得多。难读的模型与模型代码，会限制我们同同事分享实现的能力；而分享正是科学计算的一项首要目标。

最后，并非每一种模型结构与数据组合都受益于非中心化参数化。有时中心化版本——把均值与标准差放进先验——反而更好。因此可以先尝试你个人觉得最自然的形式；若遇到麻烦，再换一种。积累一些经验后，同一模型的不同形式便会渐渐熟悉。本章末有一道练习题可能有所帮助。

14.3 工具变量与因果设计

回到第 6 章，你曾接触一套判断回归中该使用哪些变量的框架。核心思想是：在 DAG 这样的图模型里，一项变量与结果之间可能有许多路径。有些路径是因果路径，我们希望让它们保持开放；另一些是非因果路径，例如后门路径，我们希望把它们关闭，同时避免因把错误变量纳入模型而意外打开它们。

当然，有时根本不可能关闭所有非因果路径。那该怎么办？仍然不是什么都做不了。若足够幸运，可以利用自然实验与巧妙建模的组合，即使无法关闭非因果路径，也能作出因果推断。

先从最著名、可能也最反直觉的例子讲起，然后再介绍其他方法。

14.3.1 工具变量

考虑教育 E 对工资 W 的影响。接受更多学校教育，会提高未来工资吗？若只是把工资对实际受教育程度作回归，推断预计会受到同时影响工资和教育的因素所偏置。例如，勤奋的人既可能完成更多教育，也可能赚得更高工资，由此产生教育与工资的相关；但这未必意味着教育会导致更高工资。这类所有潜在混杂因素往往既难测量，甚至也难以全部想象。最后会得到这样一张 DAG：

怎样在模型中使用 Q？不能像普通预测变量一样，直接把它加进回归。为什么？假设把 W 对 E 作回归——这正是我们想了解的关系，但关联受到经过 U 的后门路径混杂。若再把 Q 当作另一个预测变量加进模型，坏事就发生了。图中没有经过 Q 的后门路径，但从 Q 到 W 有一条经过 U 的非因果路径：Q → E ← U → W。这条路径不是因果性的，因为改变 Q 并不会经此路径改变 W。可是在同一个模型中以 E 为条件，而 E 又是 Q 与 U 的碰撞点，于是非因果路径被打开。这会混杂 Q 的系数；它不会是零，因为它会拾取 U 与 W 之间的关联。结果，E 的系数也可能被混杂得更严重。以这种方式使用时，Q 这样的工具可以称为偏差放大器。196

这些确实很令人困惑，来看一个例子。假设 Q 表示一个人出生在一年中的哪个季度——冬、春、夏、秋。它为什么可能影响教育？因为年初出生的人往往受教育更少：一方面，他们开始上学时生理年龄更大；另一方面，他们也会更早达到合法辍学年龄。现在，若 Q 确实只能通过 E 影响 W，且 Q 也不受混杂因素 U 影响，那么 Q 就是这种神秘的工具变量。这意味着无需测量 U，也能以一种特殊方式对 E → W 作出有效因果推断。

工具变量最难之处，是相信某个具体工具真的成立。若相信自己的 DAG，工具看起来就容易相信；但又该不该相信这张 DAG？例如，一项岛屿研究用风向作为工具，推断殖民主义对经济发展的影响。198 殖民史与经济表现受到许多因素混杂，例如岛屿的自然资源。若风向影响殖民日期——风帆时代的信风让某些岛屿更容易抵达——却不直接影响经济表现，它就能充当工具。这个想法很巧妙；但也很容易想象，风会影响岛屿的许多方面，包括殖民前的接触史和生态，而这些变量又会影响当前经济。

更常见的工具，是到某项服务的距离。例如，若想估计医疗照护对母亲福祉的影响，无法轻易把医疗照护在母亲之间随机化——首先这并不合乎伦理。但若母亲到照护中心的距离自然不同，而且这些距离相对于既有健康变量是随机的，那么距离可能成为一个工具：它影响医疗照护使用，却不直接影响健康。不过，要想到医院距离与健康影响因素相关、从而违反排除限制的方式并不难。199

一般而言，不可能从统计上证明一个变量是不是好工具。和往常一样，必须依靠数据之外的科学知识来理解数据。

再思考：两阶段最糟二乘

工具变量模型常与一种名为两阶段最小二乘（2SLS）的估计程序一同讨论。它包含两次线性回归：把第一次回归的预测值作为数据送入第二次，再作调整使标准误有意义。令人惊奇的是，条件顺利时，这个程序确实有效；但它依赖大样本近似，也有众所周知的问题。200 像所有魔像一样，只需负责任地使用它。

有时人们会把 2SLS 程序误当成工具变量模型，二者并不相同。任何模型都能用多种程序估计，每种程序各有收益和代价。2SLS 的限制很大：若结果是计数、存在测量误差或缺失值，或者需要变化效应，2SLS 就很可疑。如今已有能力更强的程序，拟合工具变量模型容易了一些，但仍可能很困难。DAG 说某个效应原则上可估计，并不保证现实中就能估出来；无论怎样近似后验，也始终很难确信工具变量确实合格。

14.3.2 其他设计

工具变量是伪装成随机实验的自然实验。在上一节示例中，出生季度 Q 就像对教育 E 的外部操纵。这种施加在教育上的外部冲击类似实验操纵：它让我们能估计外部冲击的影响，并由此导出因果估计。

寻找自然实验可能有很多方式，并非全都严格属于工具变量；但只要能相信其假设，它们就能提供理论上正确的因果推断设计。下面再看两种。

更常见的设计是回归不连续（RDD），本质上是特殊的工具变量设计。假设要估计获得学术奖项对未来成功的影响。202 能力等未观测因素同时影响奖项与后续成功，从而造成混杂；但若比较刚好低于获奖门槛与刚好高于门槛的人，他们在未观测因素上应更相似。对门槛附近的人而言，奖项仿佛随机施加——这就是回归不连续的核心。实践中，门槛上方和下方各拟合一条回归；因果效应估计就是刚好高于与刚好低于门槛者之间的平均差。虽然关心的是门槛附近差异，整个函数却会影响这个差，因此选择暴露与结果整体关系的函数时必须谨慎。203

图 14.8 展示跨二元组的原始赠礼分布，整体相关为 0.24；但把它当成交换平衡的度量会很糟。首先，交换 A/B 标签后相关会改变，而标签本来就是任意的，所以测得的相关也有些任意。其次，背景生成模型认为赠礼既可由每个二元组的特殊关系解释——有些住户彼此频繁交换——也可由某些住户不论二元组都普遍多给或多收来解释。例如贫困住户可能给得少却收得多。要在统计上把平衡交换与一般性的给予/接受差异分离，模型必须把二者分开处理。这类模型通常称为社会关系模型（SRM）。

具体而言，把住户 A 给住户 B 的礼物建模为住户专属和二元组专属变化效应的组合。结果是没有明显上界的计数，因此使用 Poisson 分布，并像前几章那样用 log 链接连接变化效应：

yA→B ∼ Poisson(λAB)
log λAB = α + gA + rB + dAB

线性模型中的截距 α 表示所有二元组的平均赠礼率（在 log 尺度上），其他效应都是相对这个平均值的偏移。gA 是住户 A 不论二元组的一般给予倾向，rB 是住户 B 不论二元组的一般接受倾向，dAB 则是 A 给 B 的二元组专属率。同一二元组中另一个方向有对应模型：

yB→A ∼ Poisson(λBA)
log λBA = α + gB + rA + dBA

合起来，每户 H 都需要两个变化效应 gH 与 rH；每个二元组 AB 还需要 dAB 与 dBA。我们既希望 g 与 r 可以相关——给得多的人是否也收得多？——也希望两个二元组效应相关——二元组内部是否平衡？两个不同的多元正态先验便能同时做到。第一个表示住户效应总体：

giri ∼ MVNormal(00, σg2σgσrρgrσgσrρgrσr2)

对任意住户 i，一对 g、r 参数都获得一个含两个标准差和一个相关参数的常规协方差矩阵先验。第二个多元正态先验表示二元组效应总体：

dijdji ∼ MVNormal(00, σd2σd2ρdσd2ρdσd2)

住户 i 与 j 组成的二元组有一对效应，它们共享另一个协方差矩阵。这个矩阵只有一个标准差 σd，因为二元组中的标签任意，谁先谁后没有意义，所以两个参数必须有相同方差；但仍需估计相关 ρd。它为正时，一户在二元组内给得更多，另一户也给得更多；为负时，一户多给、另一户少给；接近零则表示二元组内没有模式。

赠礼例子是一种社会网络模型。由此看来，当前模型缺少的一个重要特征是社会关系的传递性：若住户 A 与 B 是朋友，而 C 与 B 也是朋友，那么 A 与 C 更可能成为朋友。这不是魔法，只是未观测因素制造相关关系。例如常去同一家酒吧的人往往彼此认识；对推断社会关系成因而言，酒吧就是未测量混杂。能估计并预期传递性的模型会更好，这可用随机区块模型实现。不过拟合此类模型需要下一章的一些技术。

14.5 连续类别与 Gaussian 过程

到目前为止，所有变化效应——无论截距还是斜率——都定义在离散、无序的类别上。例如咖啡馆是独特地点，不存在咖啡馆 1 排在咖啡馆 2 前面的含义；“1”和“2”只是独特事物的标签。蝌蚪池塘、大学院系与单只黑猩猩也一样。为每个集群估计独特参数，可以量化导致集群间变异与集群内观测协变的一些独特特征；跨集群汇聚既提高准确度，也同时描绘变异。

但年龄、收入或身高这类连续变异维度呢？同龄人共享某些暴露：听过相同音乐、听说过相同政治人物、经历过相同天气事件。年龄相近的人也共享其中一部分，但程度不如完全同龄者。两个人在年龄、收入、身高或其他背景相似性维度上越不相似，协变就越弱。为所有同龄者估计一个独特变化截距、却忽略年龄相近者应有更相似截距，并不合理；而且样本中很可能每个人的年龄都不同，能利用的只有连续相似度差异。

幸运的是，变化效应方法可以推广到这种连续类别。这样既能为任意年龄估计独特截距（或斜率），又把年龄视为连续维度，使相近年龄拥有更相似的截距（或斜率）。一般方法称为 Gaussian 过程回归。205 遗憾的是，这个名字完全没有说明它用来做什么、怎样工作。

下面通过一个基本例子说明 Gaussian 过程回归能做什么、又怎样工作。一般目的，是定义一个案例彼此有差异的维度，例如个体年龄或地理位置；然后测量每对案例之间的距离。模型要做的，就是估计距离不同的案例对之间协方差的函数。这个协方差函数把变化效应方法推广到连续类别。

14.5.1 示例：大洋洲工具的空间自相关

第 11 章研究历史上大洋洲社会的工具组合复杂度时（原书 PDF 第 354 页），我们用一个粗糙的二元接触预测变量代理社会间可能存在的交换。但这个变量很不令人满意。第一，它没有说明各社会究竟与哪些社会有接触；若所有邻居都是小岛，即使接触率很高，也未必会增加多少工具复杂度。第二，工具确实会在社会间交换——这一点已知属实——那么即使以所有预测变量为条件，各社会的工具总数也并非相互独立。相反，地理上接近的邻居会因交换而有更相似的工具数。第三，相近岛屿可能共享未测量的地理特征，例如石料或贝壳来源，由此发展出相似的技术产业。因此，空间可能通过多条路径发挥作用。

这正是使用 Gaussian 过程回归的经典场景。先定义社会之间的距离矩阵，再估计工具数的相似程度如何依赖地理距离。普通预测变量也可同时放进模型，使社会随距离产生的协变既控制其他技术影响因素，也受到这些因素的控制。

决定 K 之协方差模型形状的部分是 exp(−ρ2Dij2)。Dij 是第 i 与第 j 个社会之间的距离，所以这个函数表示两社会的协方差随距离平方呈指数下降；ρ 决定下降速率。它越大，协方差随距离平方下降得越快。

为什么要把距离平方？并非必须，这只是一个模型。平方距离是最常见的假设，一方面因为容易用数据拟合，另一方面因为它常有一个现实属性：距离越大，协方差可以下降得越快。把它与线性距离下降方案 exp(−ρ2Dij) 画在一起就很直观。示例令 ρ2 = 1：

14.5.2 示例：系统发育距离

物种像岛屿一样，彼此之间有远有近。不过这种距离不是物理上的，而是时间上的：距离共同祖先已有多久？演化生物学家研究系统发育关系怎样影响不同物种身体与脑部的变异模式。共同祖先越近，物种之间的性状相关越高，这是事实；但这些相关重要吗？

系统发育距离可能产生两种重要的因果影响。第一，若性状不是由选择维持，而是在中性漂变中游走，刚分化不久的两个物种通常更相似。第二种影响是间接的：即使选择很重要，系统发育距离仍可代理那些造成物种间协变的未观测变量。近缘物种可能共享更多这类变量，远缘物种则少得多。例如所有哺乳动物都用乳汁哺育幼崽，鸟类飞行也会影响许多性状。这些离散、足以改变生活史的性状，对其他性状可能有很强的因果影响；若它们未被观测，系统发育距离就可能成为有用代理——但前提是性状模型捕捉到正确细节。206 这些方法并不会自动奏效，不能像期刊论文常做的那样仪式化地套用。

在使用树之前，先作一个普通回归：以（log）群体大小和（log）体重预测（log）脑大小。这里故意用不普通的方式构造普通回归，以便理解下一步怎样嵌入系统发育信息。把所有物种看成一个变量，即由 301 个性状值组成的向量；其中一些值当然彼此更相似。典型回归用预测变量建模这种相似性，而以预测变量为条件后，模型预期仍有相关。因此可以用一个大型多元结果分布写出模型：

B ∼ MVNormal(μ, S)
μi = α + βGGi + βMMi

其中，B 是各物种脑大小组成的向量，S 是行数、列数都等于物种数的协方差矩阵。在普通回归中，这个矩阵写成：

S = σ2I

σ 仍是从第 4 章起一直使用的标准差，I 是单位矩阵：主对角线上全为 1，其余位置全为 0。也可以把它看成所有相关都为零的相关矩阵；将方差乘进去，只会让每个物种具有相同的残差方差。这仍是普通线性回归，只是把它理解成一个多元结果。

下面进行两种系统发育回归。两者所做的事都很简单：用编码了系统发育信息的矩阵替换上面的协方差矩阵 S。第一种是最古老也最保守的方法之一：用 Brownian 运动解释系统发育，并由此导出一种非常特定的协方差矩阵。Brownian 运动就是 Gaussian 随机游走。若物种分化后，各自性状相对于彼此随机漂移，那么一对物种之间的协方差最终会与它们的系统发育枝长距离呈线性关系——距离越远，协方差按比例越小。当然，我们关心的性状显然不是中性演化的，而且在树的不同部分也以不同速率演化；遗憾的是，接下来这套方法仍是最常见的系统发育控制形式。

Brownian 运动模型是一种特殊的 Gaussian 过程，其中协方差会随着距离增加，以非常僵硬的方式下降。没有必要如此僵硬，而且有充分理由认为演化并不能由 Brownian 运动很好描述。常见做法之一是用 Pagel’s λ 修正 Brownian 模型，但它只是用同一个因子缩放所有物种相关，仍保留同样武断而不现实的距离模型。另一个常见方案是 Ornstein–Uhlenbeck 过程（OU 过程）：一种有阻尼、倾向回归某个均值（或多个均值）的 Brownian 运动。实践中，它约束了变异，使系统发育距离与协方差之间的关系变为非线性。208 更准确地说，OU 过程把物种 i 与 j 之间的协方差定义为：

K(i, j) = η2 exp(−ρ2Dij)

这是指数距离核，不同于上一例中的二次核。指数核表示点（物种）之间的协方差会快速下降，生成的函数因此远不如二次核平滑。它通常也更难拟合，因为函数粗糙得多；实践中需要格外谨慎地指定先验，可能还要把先验收窄。

OU 过程仍是 Gaussian 过程，所以拟合方式与上一节的二次核相同。系统发育回归文献尚未充分强调这一点；把模型写成 Gaussian 过程，便能按具体问题定制函数空间。209 这种框架目前仍不常见。生物学家往往笼罩在迷信与惧怕按错按钮的阴云下使用系统发育；Gaussian 过程框架既统一了既有方法，又允许模型描述协变随系统发育距离变化的模式。它也应当说明：纳入系统发育距离并不存在唯一正确的方法。若目标是估计因果效应，只拒绝某个零模型远远不够；我们必须有效重建未测混杂之间的模式，而不同的演化历史会需要不同模型。
