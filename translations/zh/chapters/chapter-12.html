第十二章

怪物与混合

当前译稿对应原书 PDF 第 397–414 页

在夏威夷传说中，Nanaue 是一条爱上人类女子的鲨鱼之子。他长成一个凶残的男人，背部中央还长着一张鲨鱼嘴。在希腊传说中，Minotaur 是牛头人身的怪物，由一名人类女子与一头大白牛所生。Gryphon 是一种半鹰半狮的传说怪物。毛利传说中的 Taniwha 则兼具蛇、鸟乃至鲨鱼的特征，很像中国与欧洲神话里的龙。

把不同生物的部件拼在一起，很容易就能造出怪物。许多怪物都是混合体，许多统计模型也是。本章要把前几章较简单的组成部分拼接起来，构造 likelihood 与 link 函数。像传说中的怪物一样，这些混合 likelihood 含有其他模型类型的部件；它们继承了各个部件的一些性质，从而帮助我们对具有棘手但常见性质的结果变量建模。既然是怪物，这些模型既强大又危险：它们通常更难估计，也更难理解。不过，只要具备一些知识并保持谨慎，它们就是重要工具。

我们会考察三类常见而有用的例子。第一类模型处理过度离散：它们扩展上一章的二项模型与 Poisson 模型，稍稍应对未测量的变异来源。第二类是一族零膨胀模型与零增广模型，每个模型都把一个二元事件同普通 GLM likelihood——例如 Poisson 或二项 likelihood——混合起来。第三类是有序类别模型，适用于具有固定顺序的类别结果。它把分类 likelihood 函数同一种特殊的 link 函数（通常是累积 link）合并起来。我们还会学习怎样构造有序类别预测变量。

这些模型让我们能够改造建模方法，以适应测量中不方便的现实，而不是改造测量去迁就模型的约束。还有许多其他模型也出于这个目的、以这种方式产生——把较简单模型的零件混合起来。我们不可能全部讲完；但以后遇到新类型时，至少会有一个理解它的框架。如果有一天需要构造自己独有的怪物，也尽管动手。只是务必用虚拟数据模拟验证它，再把模型拟合到虚拟数据上，确认能够恢复数据生成过程。

12.1 过度离散计数

前面一章（第 7 章）曾论证：基于正态分布的模型可能对极端观测过于敏感。问题不一定在于“离群值”是坏数据，而在于过程往往是变量的混合，因而会产生更厚的尾部。假设薄尾的模型——例如纯高斯模型——很容易过度兴奋；改用 Student-t 之类的分布，有助于得到更好的推断和样本外预测。

计数模型也是如此。当计数来自不同过程的混合时，变异可能比纯计数模型预期的更大——尾部更厚。这同样会让模型过度兴奋。当计数的变异大于纯过程所允许的程度时，就表现出过度离散。变量的方差有时也称为它的离散度。对二项过程这样的计数过程，方差与期望值由同一组参数决定。例如，二项分布的期望值是 Np，方差是 Np(1 − p)。若在以所有预测变量为条件之后，观测方差仍超过这个量，就说明某个遗漏变量正在给观测计数带来额外离散。

这不一定是坏事；这样的模型仍可能产生完全良好的推断。但忽略过度离散，也可能带来忽略任意预测变量时会出现的所有问题。计数中的异质性可能形成混杂，掩盖感兴趣的效应，或产生虚假推断。因此，值得设法处理过度离散。最好的办法当然是找到被遗漏的离散来源，把它纳入模型；不过，即使没有其他可用变量，也能缓解过度离散的影响。下面考察两种常见而有用的策略。

本章先考察连续混合模型：线性模型不直接连接到观测本身，而是连接到观测的一个分布。本节余下部分会勾勒这类模型，并使用常见的 beta-binomial 与 gamma-Poisson（负二项）模型。上一章末尾已经提到它们，现在则真正给出定义。

后面几章会介绍多层模型：它们同时估计每个观测的残差及这些残差的分布。实践中，用多层模型（GLMM，第 13 章）替代连续混合通常更容易，因为多层模型灵活得多，能同时处理过度离散与其他形式的异质性。

12.1.1 Beta-binomial

Beta-binomial 模型假设，每个二项计数观测都有自己独特的成功概率。175模型估计的是各案例成功概率的分布，而不是单一的成功概率；预测变量改变的也是这个分布的形状，而不是直接决定每一次成功的概率。

结合例子会更容易理解。上一章见过的 UCBadmit 数据，只要忽略院系，就表现出很强的过度离散，因为各院系的基准录取率差异很大。你已经看到，忽略这种变异会对申请者性别作出错误推断。现在拟合一个忽略院系的 beta-binomial 模型，看看它怎样捕捉由遗漏变量造成的变异。

对这些数据使用 beta-binomial 模型，意味着假设数据表每一行的观测计数都有一个独特、未观测的录取概率；这些录取概率本身又来自同一个分布。这个共同分布用 beta 分布描述——它是一种“概率的概率分布”。为什么用 beta 分布？因为这样数学上很方便：采用 beta 分布时，可以推导出一个封闭形式的 likelihood 函数，对每个观测未知的概率完成平均。细节见本节末尾原书第 387 页的“过度思考”框。

Beta 分布有两个参数：平均概率 p̄ 与形状参数 θ。176形状参数 θ 描述分布有多分散。当 θ = 2 时，从 0 到 1 的每个概率都同样可能。θ 大于 2 并继续增大时，概率分布会越来越集中；当 θ < 2 时，分布非常分散，以至于接近 0 与 1 的极端概率比均值附近更可能。可以摆弄参数，体会这个分布能呈现哪些形状：

12.1.2 负二项或 gamma-Poisson

负二项模型，更有用的名称是 gamma-Poisson 模型，它假设每个 Poisson 计数观测都有自己的率。177模型估计一个 Gamma 分布的形状，用它描述各案例的 Poisson 率；预测变量调整的是这个分布的形状，而不是每个观测的期望值。Gamma-Poisson 模型与 beta-binomial 模型非常相似，只是用率（或期望值）的 Gamma 分布替代成功概率的 beta 分布。为什么是 Gamma？仍然因为数学上方便：把 Poisson 概率与服从 Gamma 分布的率混合起来时，存在简单的解析表达式。Gamma-Poisson 模型非常有用，因为 Poisson 分布非常窄——还记得，它的方差必须等于均值。

Gamma-Poisson 分布有两个参数：一个控制均值（率），另一个控制各案例率的离散度（尺度）：

yi ∼ Gamma-Poisson(λi, ϕ)

参数 λ 可以像普通 Poisson 的率一样处理；参数 ϕ 必须为正，并控制方差。Gamma-Poisson 的方差是 λ + λ2/ϕ，所以 ϕ 越大，分布就越像纯 Poisson 过程。

用上一章的大洋洲工具例子看看它怎样工作。那里有一个高影响力点 Hawaii；在对应的 gamma-Poisson 模型中，它的影响力会小得多。为什么？因为 gamma-Poisson 预期均值率周围有更多变异，所以 Hawaii 对回归趋势的拉动会减弱。

新的 gamma-Poisson 模型对原始数据的后验预测显示在图 12.2；旁边放着上一章的纯 Poisson 模型 m11.11 作为比较。Hawaii 在纯 Poisson 模型中影响力很高，几乎独自把低接触率趋势向下拉。在新模型中，Hawaii 仍有影响力，但对趋势的作用小得多；高、低接触率趋势现在相似得多，很难可靠地区分。原因是 gamma-Poisson 模型预期率会变化，而估计出的变异相当大。人口仍与工具总数强烈相关，但接触率的影响已经大幅减弱。

12.1.3 过度离散、熵与信息准则

Beta-binomial 与 gamma-Poisson 模型，分别在与普通二项分布和 Poisson 分布相同的约束下达到最大熵。它们只是试图解释概率与率中未观测的异质性。因此，尽管这些模型拟合起来可能困难得多，却仍可像普通二项与 Poisson GLM 那样理解。从使用信息准则比较模型的角度看，beta-binomial 模型是二项模型，gamma-Poisson（负二项）模型则是 Poisson 模型。

不过，除非非常清楚自己在做什么，否则不要对这些模型使用 WAIC 与 PSIS。普通二项模型和 Poisson 模型可以把数据行聚合或拆开，而不改变任何因果假设；beta-binomial 与 gamma-Poisson 模型却并非如此。原因是，beta-binomial 或 gamma-Poisson likelihood 会给数据的每一行分配一个未观测参数。接下来计算对数 likelihood 时，数据的组织方式会决定服从 beta 分布或 Gamma 分布的变异怎样进入模型。

例如，本章前面考察的 beta-binomial 模型，每一行都包含计数。当时各行是院系与性别的组合，并假定同一院系—性别组合中的所有申请者共享一个未知的基准录取概率。我们真正想做的是把每份申请都视为一项观测，在申请层面计算 WAIC，从而估计一份新申请投向已知院系—性别组合时的准确度。确实可以把数据拆开，让每一行只包含一份申请；但这样就丢掉了 beta-binomial 模型所蕴含的信息：原数据同一行中的所有申请者共享同一个潜在概率。这非常麻烦。

怎么办？下一章会介绍怎样用多层模型纳入过度离散，到那时这个障碍就会减轻。为什么？因为多层模型可以在任意聚合层级上为概率或率分配异质性。

过度思考：连续混合

Beta-binomial 这样的分布称为连续混合，因为模型假定每个二项计数都有自己独立的、服从 beta 分布的成功概率，而 beta 分布是连续的，不是离散的。因此，beta-binomial 的参数只是每个案例中的抽取次数（与普通二项分布的“规模” n 相同），以及描述 beta 分布形状的两个参数。由此可得，从 beta-binomial 过程中观察到 y 次成功的概率为：

f(y | n, p̄, θ) = ∫01 g(y | n, p)h(p | p̄, θ) dp

其中，f 是 beta-binomial 密度，g 是二项分布，h 是 beta 密度。上面的积分和应用概率中的多数积分一样，只是在计算一个平均值：把 y 的概率对 p 的所有取值取平均。p 值来自均值为 p̄、尺度为 θ 的 beta 分布。成功概率 p 不再是自由参数，因为它由 beta 分布产生。Gamma-Poisson 密度也有类似形式，只是把 Poisson 概率对率的 Gamma 分布取平均。

对 beta-binomial 与 gamma-Poisson 来说，上面的积分都可以化为封闭形式。需要解析形式时，随时可以查阅这些封闭表达式；R 函数 dbetabinom 与 dgampois 已经用它们提供了计算。

12.2 零膨胀结果

我们能够测量的事物，往往并不是某种纯过程的产物，而是多个过程的混合。只要同一种观测可能由不同原因造成，混合模型就可能有用。混合模型使用不止一个简单概率分布，对多种原因的混合建模；实际上，这类模型会为同一个结果变量使用不止一个 likelihood。

计数变量尤其容易需要混合处理，因为零计数往往不止一种来路。“零”表示什么也没发生；而什么也没发生，可能是因为事件发生率很低，也可能是因为产生事件的过程根本没有启动。假如我们在林地里数灌丛松鸦，记录到零，既可能是林地中没有灌丛松鸦，也可能是我们在开始观察前就把它们全吓跑了。不论哪种情况，数据里都出现一个零。

本节将展示怎样构造简单的零膨胀模型。仍然可以使用早先模型中的相同部件，只是装配方式不同。因此，即使永远不需要使用或解释零膨胀模型，看看它们如何构造，也应当能够拓展你的建模想象力。

再思考：打破定律

科学界有时会形成一种围绕统计推断的焦虑文化。过去，研究者很难自行构造和研究自定义模型，因为必须先依赖统计学家把模型研究妥当。这便引发了对非常规模型的担忧，仿佛它们会违反统计学定律。如今，统计计算能力已经强大得多。你可以设想自己的生成过程，从中模拟数据，写出模型，再验证它能否恢复真实参数值。你不必等一位数学家来把所需模型“合法化”。

12.2.1 示例：零膨胀 Poisson

第 11 章用修道院制作手稿的例子介绍了 Poisson GLM。每天，大量僧侣各自完成抄写少量手稿。这个过程本质上是二项过程；但因为试验次数很多、概率极低，所以分布趋近 Poisson。

现在设想，僧侣有些日子会休息。这些日子里没有手稿完成；酒窖反而会打开，大家从事一些更世俗的享乐。身为修道院主人，你想知道僧侣多常饮酒。推断的障碍在于：即使在诚实工作、没有饮酒的日子里，也可能纯属偶然地出现零。那么，怎样估计饮酒天数呢？

我们来构造一个混合模型解决这个问题。178要考虑数据里的任意一个零都可能来自两个过程：（1）僧侣整天饮酒；（2）僧侣当天工作，却没能完成任何手稿。令 p 表示僧侣整天饮酒的概率，令 λ 表示他们工作时完成手稿的平均数量。

要启动这个模型，需要定义一个把两个过程混在一起的 likelihood 函数。为了理解怎样构造这种怪物，可以把僧侣是否饮酒想成一次抛硬币的结果（图 12.3）。“硬币”一面画着酒桶，另一面画着羽毛笔。酒桶朝上的概率是 p，可以取 0 到 1 之间的任意值。根据抛硬币的结果，僧侣要么开始饮酒，要么开始抄写。饮酒的僧侣完成的手稿数总是零；工作的僧侣完成的手稿数则服从均值率为 λ 的 Poisson 分布。因此，即使僧侣工作，也仍可能观察到零。

在这些假设下，观察到零的 likelihood 是：

Pr(0 | p, λ) = Pr(饮酒 | p) + Pr(工作 | p) × Pr(0 | λ)
= p + (1 − p) exp(−λ)

由于 y 的 Poisson likelihood 是 Pr(y | λ) = λy exp(−λ)/y!，当 y = 0 时，likelihood 就只是 exp(−λ)。上式只是在用数学表达：

观察到零的概率，等于僧侣饮酒的概率，或（+）僧侣工作且（×）什么也没完成的概率。

原文表述不一致：上面的公式与模型设定都令 p 为饮酒概率，原书紧随公式的解释句却印作 “the monks didn’t drink”。译文按公式与全文含义改为“饮酒”，并在此记录原文差异。

非零值 y 的 likelihood 则是：

Pr(y | y > 0, p, λ) = Pr(饮酒 | p)(0) + Pr(工作 | p) Pr(y | λ) = (1 − p) λy exp(−λ)/y!

因为饮酒的僧侣永远不会产生 y > 0，上式只是僧侣既工作——概率为 1 − p——又完成 y 份手稿的概率。

把上面这个分布定义为 ZIPoisson，用参数 p（产生零的概率）与 λ（Poisson 均值）描述它的形状。那么，零膨胀 Poisson 回归可写成：

yi ∼ ZIPoisson(pi, λi)
logit(pi) = αp + βpxi
log(λi) = αλ + βλxi

注意，ZIPoisson 中每个过程各有一个线性模型和一个 link 函数，一共两套。两个线性模型的参数不同，因为 x 这样的任意预测变量，可能与混合模型的两个部分呈现不同关联。事实上，两个模型甚至不必使用相同的预测变量——可以根据假说，任意构造这两个线性模型。

if ( y[i] > 0 ) target += log1m(p) + poisson_lpmf(y[i] | lambda);
if ( y[i] == 0 ) target += log_mix(p, 0, poisson_lpmf(0 | lambda));

target 是计算对数后验的一串项；用 += 时，就是把另一项加入这叠计算。随后 Stan 通过积极而系统地运用微积分链式法则，利用这串项计算梯度。这里还有几个重要的计算技巧。函数 log1m 计算 1 减某个值之后的对数。我们需要 log(1 − p)；但当 p 非常接近 1 时，直接相减可能灾难性地舍入为零，随后对数就会变成负无穷。使用 log1m 能大大降低这种风险。

函数 log_mix 把两个对数概率混合起来，正好用于零的概率；它也采用巧妙方法避免舍入误差。在这个例子里，它等价于：

if ( y[i] == 0 ) target += log( p + (1-p)*exp(-lambda) );

但在 p 取极端值时更加稳定。本例中两种写法没有区别，朴素的直接方法也能正常工作；但值得了解更好的做法。你会在代码里反复见到它，否则更复杂的模型可能无法正确运行。最后，请注意数据列表里把 y 强制转换成了整数。使用 ulam 的内置分布时，它会尝试把变量转换为正确的 Stan 类型；如果自行构造分布，就需要自己完成转换。

12.3 有序类别结果

在社会科学中非常常见、在自然科学中偶尔也会出现这样的结果变量：它像计数一样是离散的，但数值只表示某个维度上不同的有序层级。例如，假如我问你有多喜欢吃鱼，请在 1 到 7 的尺度上作答，你可能回答 5。若对 100 个人提出同样的问题，最终会得到 100 个介于 1 与 7 之间的值。对每个结果值建模时，必须记住这些值有顺序，因为 7 大于 6，6 又大于 5，依此类推。结果就是一组有序类别。与计数不同，数值之间的差异并不一定相等。让某人对鱼的偏好从 1 变为 2，可能远比从 5 变为 6 困难。把有序类别直接当作连续测量并不是好主意。179

幸运的是，这个问题有一种标准而易用的解法。原则上，有序类别变量只是一个多项预测问题（原书第 366 页）；但类别必须有序这一约束，要求特殊处理。我们希望任何相关预测变量随着自身增大，都能让预测依次、渐进地穿过各个类别。例如，如果冰淇淋偏好与年龄呈正关联，那么模型应当随着年龄增加，让预测依次上移：从 3 到 4、从 4 到 5、从 5 到 6，等等。这就带来一项挑战：怎样保证线性模型按正确顺序映射到结果？

传统解法是使用累积 link 函数。180某个值的累积概率，是该值本身或任意更小值的概率。在有序类别中，3 的累积概率就是 3、2、1 三个值的概率之和。按照惯例，有序类别从 1 开始，因此小于 1 的结果完全没有概率。把线性模型连接到累积概率，就能保证结果的顺序。

下面分两步解释原因。第一步说明怎样在对数累积赔率尺度上参数化结果分布。第二步把一个（或多个）预测变量引入这些对数累积赔率，从而在服从有序预测性质的同时，对预测变量与结果之间的关联建模。

为了让讨论更具体，两步都会放在一个数据例子中展开。下面先认识数据。

12.3.1 示例：道德直觉

这个例子的数据来自哲学家开展的一系列实验。181没错，哲学家有时确实会做实验。这些实验旨在收集与道德直觉争论有关的经验证据；道德直觉是人们据以判断行为在道德上善恶的推理形式。这类争论与所有社会科学都有关，因为它们触及更广泛的推理问题、情绪在决策中的作用，以及个人与群体的道德发展理论。

这些实验用所谓的“电车难题”情景测量道德判断。经典版本涉及一辆失控电车；这些情景的共同点，是都会令道德哲学家困惑，甚至感到悖谬。下面是一个传统例子，只是用“空货车”代替了“电车”：

Dennis 站在铁轨旁，看见一辆空的失控货车即将撞上五个人。Dennis 身边有一根拉杆；拉动它，就会让货车转入侧轨，避开那五个人。但拉动拉杆也会放下跨越侧轨的人行桥护栏，使一个人从桥上跌落到侧轨，随后被货车撞上。如果 Dennis 拉动拉杆，货车就会换轨，不再撞上那五个人，但会撞上跌落的那一个人。如果 Dennis 不拉，货车就会沿原轨继续行驶，撞上五个人，而另一个人则安全地留在侧轨上方。

从道德上看，Dennis 拉动拉杆有多大程度是可以允许的？

这类情景之所以在哲学上令人困惑，是因为两个情景的分析内容可能完全相同，人们却会对不同情景中同一个行为的道德可允许性，稳定地作出不同判断。在你急着断定这些东西很荒唐之前，请注意：人们在这些实验中的道德直觉，与他们面对法律以及行为是否应被归类为犯罪时的反应很相似。法律中充满了道德悖论；我们需要理解它们。

既有研究至少提出了三条重要的无意识推理原则，或许能解释判断差异：

  • 行动原则：主动行为造成的伤害，在道德上比不作为造成的等量伤害更糟。
  • 意图原则：作为实现目标之手段而有意造成的伤害，在道德上比作为目标副作用而预见到的等量伤害更糟。
  • 接触原则：通过身体接触伤害受害者，在道德上比不使用身体接触造成的等量伤害更糟。

我们探索这些原则的实验语境，由一组故事构成：故事改变原则，却让许多基本物体与行动者保持不变。例如，上面引用的空货车故事包含行动原则，却不包含另外两条。行动者 Dennis 必须做点什么才能改变结果，而不是保持被动，所以这是行动情景。然而，为了救下五个人，并不需要、也不打算让会跌落的那个人受到伤害，因此它不是意图原则的例子。故事也没有直接接触，所以同样不是接触原则的例子。

可以按同样的故事轮廓，再构造一个同时包含行动原则与意图原则的空货车故事。也就是说，在这个版本里，行动者既要采取行动改变结果，这个行动又必须伤害一个人，才能救下另外五个人：

Evan 站在铁轨旁，看见一辆空的失控货车即将撞上五个人。Evan 身边有一根拉杆；拉动它，会放下跨越主轨的人行桥护栏，使一个人从桥上跌落到主轨，随后被货车撞上。货车会因为撞上这个人而减速，从而避免撞上那五个人。如果 Evan 拉动拉杆，那一个人会跌落并被货车撞上，于是货车减速，不再撞到五个人。如果 Evan 不拉，货车就会沿铁轨继续行驶，撞上五个人，而另一个人则安全地留在主轨上方。

多数人判断，如果 Evan 拉动拉杆，这在道德上会比 Dennis 拉动拉杆更糟（更不允许）。分析这些数据时，你会看到差异究竟有多大。先载入数据：

我们的目标，是在对数累积赔率尺度上重新描述这个直方图。做法就是先构造累积概率的赔率，再取对数。为什么要做这种晦涩的变换？因为它是前几章所用 logit link 的累积版本。Logit 是对数赔率，累积 logit 则是对数累积赔率。两者都用于把概率约束在 0/1 区间。以后决定加入预测变量时，就可以安全地在累积 logit 尺度上操作；link 函数会负责把参数估计转换到正确的概率尺度。

接下来，要把直方图重新描述为对数累积赔率，需要一系列截距参数。每个截距都位于对数累积赔率尺度，代表每个结果的累积概率。这只是 link 函数的应用。回答值 yi 小于或等于某个可能结果值 k 时，其对数累积赔率为：

log [ Pr(yi ≤ k) / (1 − Pr(yi ≤ k)) ] = αk(12.1)

其中，αk 是每个可能结果值 k 独有的“截距”。可以直接计算这些截距参数：

这些值绘制在图 12.4 右侧。注意，最大回答值 7 的累积 logit 是无穷大，因为 log(1/(1 − 1)) = ∞。最大回答值的累积概率总是 1，所以实际上不需要为它设置参数；根据全概率法则，它是免费得到的。因此，对 K = 7 个可能回答值，只需要 K − 1 = 6 个截距。

以上都很不错，但我们真正想要的是这些截距的后验分布。这样才能考虑样本量与先验信息，还能加入预测变量（下一小节）。要用 Bayes 定理计算这些截距的后验分布，需要求出每个可能回答值的 likelihood。因此，构造有序类别结果基础模型拟合引擎的最后一步，是用累积概率 Pr(yi ≤ k) 计算 likelihood Pr(yi = k)。

图 12.5 展示了具体做法。每个截距 αk 都为每个 k 隐含一个累积概率；只需使用逆 link，把对数累积赔率转换回累积概率。因此，观察到 k 并需要它的 likelihood 时，可以通过相减得到：

pk = Pr(yi = k) = Pr(yi ≤ k) − Pr(yi ≤ k − 1)(12.2)

图 12.5 的蓝色线段就是通过相减算出的 likelihood。有了它们，便可以照常计算后验分布。

下面看看怎样把它写成模型。有序 logit 的数学表达惯例差异很大；本书采用：

Ri ∼ Ordered-logit(ϕi, κ) 数据的概率
ϕi = 0 线性模型
κk ∼ Normal(0, 1.5) 每个截距的共同先验

也可以把模型写得更直白：

Ri ∼ Categorical(p) 数据的概率
p1 = q1 每个值 k 的概率
pk = qk − qk−1，其中 K > k > 1
pK = 1 − qK−1
logit(qk) = κk − ϕi 累积 logit link
ϕi = 线性模型中的各项 线性模型
κk ∼ Normal(0, 1.5) 每个截距的共同先验

原文下标不一致：最大类别概率应为 pK = 1 − qK−1；原书对应行把最后一个下标印成小写 k − 1。译稿按一般式改为 K − 1，并在此记录。

第二种写法很残酷，却揭示了有序 logit 分布其实只是一个类别分布：它接收向量 p = {p1, p2, p3, p4, p5, p6}，其中包含最大回答值以下各档回答的概率（本例最大值为 7）。向量中每个回答值 k，都由它与截距参数 αk 的 link 定义。最后，再给这些截距设置弱正则化先验。本例数据很多，几乎任何先验都会被数据压倒；像往常一样，在小样本情境下，必须更认真地考虑先验。例如，在看到数据之前，我们就知道 α1 < α2。

12.3.3 加入预测变量

这一通计算到目前为止几乎没带来什么，除了把一张直方图表示成 Bayesian 模型。不过，为了给模型加入服从结果有序约束的预测变量，所有这些准备都是必要的。

要纳入预测变量，就把每个回答 k 的对数累积赔率定义为其截距 αk 与普通线性模型之和。例如，假设要把预测变量 x 加入模型，可以定义线性模型 ϕi = βxi。于是，每个累积 logit 都变为：

log [ Pr(yi ≤ k) / (1 − Pr(yi ≤ k)) ] = αk − ϕi
ϕi = βxi

这种形式自动保证结果值顺序正确，同时仍能随着预测变量 xi 改变，调整每个单独值的 likelihood。为什么从每个截距中减去线性模型 ϕ？因为降低最大值以下每个结果值 k 的对数累积赔率，必然会把概率质量向上推到更高结果值。于是，β 为正就表示 x 增大时，y 的均值也增大。也可以把 ϕ 加上去，写成 αk + ϕi；但那样一来，β > 0 反而表示 x 增大时均值下降。
