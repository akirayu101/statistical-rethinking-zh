第十六章

广义线性疯狂

当前译稿对应原书 PDF 第 553–564 页

高中时，我问物理老师怎么看统计学，她给我讲了一个笑话。以下是我记得的版本。一位物理学家、一位工程师和一位统计学家结伴去用弓打猎。几个小时后，他们终于在远处发现一只鹿。物理学家快速做了弹道计算，忽略空气阻力；箭飞得很正，却落在目标前几米，鹿毫无察觉。工程师冷笑一声，给空气阻力加入一个修正因子，然后放箭；第二支箭却落在目标后几米，鹿仍然没有察觉。统计学家取了平均值，高喊：“我们射中了！”

科学会构造自然过程的理论，这些理论最终以数学模型的正式形式表达。这样的模型高度专门化，能作出精确预测，也能以同样精确的方式失败。精确地犯错很有用，因为失败会从模型内置的因果关系中获得意义。笑话里的物理学家与工程师正是如此：他们错得很精确，因此也留下线索，告诉我们哪些原因出了问题。

应用统计学必须适用于所有科学，所以它对模型往往含糊得多，转而关注平均表现，而不管模型是什么。前几章的广义线性模型，并不是多数自然过程可信的科学模型；它们是强大的、地心式的关联描述（第 4 章）。然而只要与因果推断逻辑结合——例如 DAG 与 do 演算——广义线性模型仍会强大得不讲道理。

不过，“GLM 加 DAG”也有问题。并非一切都能建模为 GLM——把变量的线性组合映射到非线性结果上；但若你只会这一种方法，就只能硬用它。有时，感兴趣的理论确实可以表达为 GLM，却同时意味着某些参数必须固定在特殊值上；若从 GLM 而不是真实模型出发，我们可能永远注意不到这一点。而当 GLM 失败时，也很难从失败中学习。调试地心体系的本轮游戏，谁也赢不了。若能用真正的结构因果模型取代启发式 DAG，也许可以一次解决所有这些问题。

本章将超越广义线性疯狂。我会逐一研究一些示例，其中科学情境提供因果模型，为统计模型注入生命。我特意选择彼此不同的例子，用来突出开发因果模型、并把它们转化成定制统计模型时的不同挑战（见下方“再思考”框）。理解这些例子不需要任何专门科学知识。基本策略仍然与全书开头相同：定义现象的生成模型，再用它设计因果推断与统计估计策略。

与本书其他章节不同，本章包含一些无法回避的数学；但只需要代数。与其说我们要“做数学”，不如说要用数学表达思想。由于某些示例对 ulam() 来说不够灵活，我们还会直接使用 Stan 模型代码。若对代码不感兴趣，可以跳过；不过一如往常，看到实现细节往往有助于澄清概念。

再思考：量身定制

批量生产有一些好处，却也让衣服不那么合身。从货架上直接买来的成衣，并不是为你而做；它们不是针对某个特定身体的定制产品。除非你幸运地拥有完美的平均体形，否则想要更合身，就得找裁缝。

统计分析也一样。广义线性模型是货架产品，为目标各异、没有耐心的研究者构成的消费市场批量生产。科学向统计学家索要一种到处都能用的工具，于是统计学家交货了；但衣服并不总合身。

货架模型的一个问题，是它会截断专业知识。典型研究者对自己的主题知道很多；当另一个专业的人试图为该主题建立理论模型时，科学家会提出细致反驳，这正是证据。但轮到分析自己的数据时，他们却使用禁止运用这些知识的工具；标准 GLM 根本没有地方容纳它们。更糟的是，若研究者学过的模型只有 GLM（或 GLMM），这些模型可能挤掉知情、定制科学模型的形成空间。

GLM 强大得不讲道理，但应当记住，它们通常只是地心式装置。最终，为了更好的拟合与推断，都需要更好的定制模型。

16.1 几何人类

回到第 4 章，你曾在用体重建立身高预测模型的情境中接触线性回归，也看过怎样度量两个变量之间的非线性关联。但那个例子在科学上毫不令人满意。身高—体重模型只是一件统计工具，不含任何生物学信息，也没有告诉我们身高与体重之间的关联如何产生。比如，体重显然不会导致身高——至少对人类不是如此；真要说的话，因果关系反而朝另一个方向。

所以现在尝试做得更好。为什么？因为模型若由科学启发，而不只是统计上需要，那么模型与数据之间的不一致，就会携带真实因果关系的信息。

例如，假设一个人长得像圆柱体。当然，人并不完全像圆柱体：还有手臂和脑袋。但先看看这套圆柱模型能走多远。圆柱的重量来自圆柱体积，圆柱体积又来自圆柱高度和宽度的增长。因此，若能把身高与体积联系起来，就会得到一个用身高预测体重的模型。

16.1.1 科学模型

开始吧。很久以前，你学过、又很明智地忘掉了：人体圆柱的体积公式是：

V = πr2h

其中 r 是这个人的半径，h 是身高，见图 16.1。219 我们不知道每个人的半径，但先假定每个人的半径都是其身高的某个固定比例 p。

也就是说，r = ph。把它代入体积公式：

V = π(ph)2h = πp2h3

最后，体重是体积的某个比例——每立方厘米对应多少千克？因此还需要参数 k，表达从体积到体重的换算：

W = kV = kπp2h3

这就是给定一个人的身高 h 时，预期体重的公式。它显然不是普通的广义线性模型，但没关系：它能作出预测，也能拟合到数据。

再思考：球形奶牛

有用的数学建模通常包含荒唐的假设，例如上面假定人类长得像圆柱体。这类假设可称为球形奶牛，名称来自《Consider a Spherical Cow: A Course in Environmental Problem Solving》。220策略性的简化假设，是所有有用模型的特征。先理解简化模型，随后就更容易在适当位置加入相关细节；简化模型暴露的缺陷，也会帮助我们判断哪些细节真正相关。非数学模型同样会简化，只是通常不把简化明确说出来，因此更难识别其缺陷。221

16.1.2 统计模型

可以在统计模型中使用圆柱公式；不过还要再作一些选择。下面先给出模型轮廓，随后逐项解释：

Wi ∼ Log-Normal(μi, σ)[体重的分布]
exp(μi) = kπp2hi3[体重的预期中位数]
k ∼ 某个先验[体重与体积之间的先验关系]
p ∼ 某个先验[半径与身高比例的先验]
σ ∼ Exponential(1)[老朋友 σ]

从顶部开始，首先要决定观测结果变量——体重 Wi——的分布。这个变量为正且连续：体重不可能为负。因此我选择了对数正态分布。对数正态分布用对数的均值参数化，这个均值称作 μi；其原尺度中位数为 exp(μi)。

在上面的模型中，我把这个中位数指定为圆柱函数。最后，还需要为 k、p 与 σ 三个参数指定先验。

科学启发模型的一项主要优势，是参数拥有意义。这些意义本身就是先验信息，可以用来选择信息性分布。在这里尤其有用，因为科学上需要的参数，往往比数据能够直接识别的参数更多；只要对参数施加一些科学约束，我们仍然可以开展有用的估计。本例正是如此。

首先注意，参数 k 与 p 在模型中彼此相乘，数据除了它们的乘积，什么也估计不了。技术上说，在给定这个模型与这些数据时，k 和 p 不可识别。可以直接用新参数 θ 替代乘积 kp2，转而估计 θ：

exp(μi) = πθhi3

这样会得到相同预测，却失去一种方便地为 θ 指定先验的方法。因此，即便打算使用 θ = kp2 这个技巧，仍需思考 k 与 p。

先想参数 p。它是半径与身高之比，p = r/h，所以必须大于零；它也必须小于一，因为很少有人比自己身高还宽。它几乎肯定小于二分之一：若一个人的宽度与身高相同，就有 2r = h，因此 p = (h/2)/h = 0.5。所以 p 大概远小于 0.5。综合起来，我们需要一个以 0 和 1 为界、且大部分先验质量位于 0.5 以下的分布。Beta 分布可以做到：

p ∼ Beta(2, 18)

这个先验的均值为 2/(2 + 18) = 0.1。若要做得更好，确实应当进行先验预测模拟（见本章末练习）；但暂时先这样处理 p。

参数 k 是体重占体积的比例。它其实只是换算度量尺度，因为改变体积或体重的单位，就会改变它的值。例如，若身高以厘米计、体重以千克计，则体积单位为 cm3，k 的单位就必须是 kg/cm3。此时 k 的定义，正是每立方厘米对应多少千克。因此，为了把先验放到正确尺度上，需要知道一立方厘米人体大约有多重。可以查资料，也可以用自己的身体建立先验。

再思考：先验从不任意

常听人忧心忡忡地宣称，Bayes 不可信，因为先验是任意的。确实，有些人会任意对待先验；但只有在科学家忽视领域知识时，先验才会任意。即使坚持使用 GLM，先验预测模拟也会迫使我们接触背景知识，从而产生有用而非任意的先验。模型的科学基础越扎实，参数意义就越丰富。圆柱示例中的 p 与 k 都有科学含义，因而可以指定甚至能够直接物理测量的先验。出于某种形而上的“无知”承诺而在本例使用平坦先验，将是个错误。

新变量 w 与 h 的均值都是 1。这里使用均值并无特殊之处，只需某个参照值来约掉单位。现在考虑在这种缩放下，k 的合理取值。假设有一位身高和体重都恰好等于平均值的人，此时 wi = 1 且 hi = 1。代入公式：

16.1.3 伪装的 GLM

进入下一个示例之前，考虑把体重的对数与身高关联起来时，这个模型会怎样变化。此时，期望为：

log wi = μi = log(kπp2hi3)

由于乘法在对数尺度上变成加法，可以改写为：

log wi = log(k) + log(π) + 2 log(p) + 3 log(hi)

在对数尺度上，这就是线性回归。前三项共同组成截距；3 log(hi) 则是系数固定为 3 的预测变量。这个系数的值由理论给出，不必估计，却仍保持普通线性回归项的形式。

指出这一点，是为了强调广义线性模型如此强大的一个原因：在某种特定度量尺度上，许多自然关系都是 GLM 关系。与此同时，GLM 路径倾向于直接估计那些本可由适当理论提供信息的参数，本例正是如此。

16.2 隐藏的心智与观测行为

科学推断最基本的问题之一，是所谓的逆问题：怎样从观测反推出原因。之所以是个问题，是因为许多不同原因可以产生相同证据。因此，从已知原因向前推导预测观测可能很容易，从观测反向追溯原因却可能非常困难。

每个科学分支都有自己的逆问题。本节考察发展心理学中的一个简单示例。儿童作决定时，可能拥有许多不同认知策略；给定一些行为观测，究竟是哪种策略造成的？具体考虑这样一项实验：629 名 4 至 14 岁儿童观看另外四名儿童在三只不同颜色的盒子中作选择（图 16.3）。随后，每名儿童自己选择一次。每轮试验中，三名示范者选择同一种颜色，第四名选择另一种颜色；于是三种颜色中，一种是多数选择，一种是少数选择，最后一种无人选择。怎样从实验判断儿童是否受到多数人的影响？

16.2.1 科学模型

关键一如往常，是从生成角度思考。例如，有一群儿童，其中一半随机选择，另一半跟随多数。模拟这些儿童的选择，就能判断我们会多常看到代表多数颜色的“2”号选择。

大约三分之二的选择落在多数颜色上，但真正跟随多数的儿童只有一半。上面只模拟了一次，却已展示问题所在：不同隐藏策略能产生相同行为时，对策略的推断远比单纯统计行为复杂。

下面考虑儿童可能使用的五种不同策略：

  1. 跟随多数：复制多数示范者选择的颜色。
  2. 跟随少数：复制少数示范者选择的颜色。
  3. 特立独行：选择没有任何示范者选择的颜色。
  4. 随机：忽略示范者，随机选择一种颜色。
  5. 跟随第一：复制最先示范的颜色；当 majority_first 等于 1 时，它是多数颜色，等于 0 时则是少数颜色。

每种策略都对应一个含三个概率的向量，三个选择各一个。例如“随机”是 [1/3, 1/3, 1/3]。较复杂的是“跟随第一”，因为它取决于呈现顺序。

一个显而易见的问题是：为什么选择这几种策略？同样显而易见的回答是：因为它们在先验上看起来合理。如果你认为其中某些不合理，或者觉得还应加入更多策略，同一个生成框架都能容纳。

16.2.2 统计模型

现在需要一个反映上述生成模型的统计模型。记住，统计模型的运行方向与生成模型相反：在生成模型中，假定策略并模拟观测行为；在统计模型中，则假定观测行为（数据）并模拟策略（参数）。

本例无法直接测量每名儿童的策略，它是未观测变量。但每种策略产生每种选择的概率都是特定的。可以利用这个事实，在用参数指定各策略概率后，计算每种选择的概率；随后放出 Bayes，取回每种策略的后验分布。不过在放出 Bayes 之前，还要列举参数、为每个参数指定先验，并解决若干编码技术问题。下面逐步完成。

未观测变量，是儿童采用五种策略各自的概率。这意味着五个数；但由于它们必须和为 1，只需四个参数。有一种叫作单纯形的变量类型能替我们处理：单纯形是一个数值向量，各值之和必须等于某个常数，通常为 1。Stan 允许把参数向量声明成单纯形，并自动管理“和固定”这项记账工作。可以给单纯形指定 Dirichlet 先验，也就是概率分布的先验。第 12 章构造有序类别预测变量时，已经用过 Dirichlet 与单纯形。这里对策略概率的单纯形使用弱均匀先验，并将其记作 p：

p ∼ Dirichlet([4, 4, 4, 4, 4])

正如第 12 章所见，这个先验并不表示我们预期各策略等概率；它表示任何一种策略都可能比任何另一种更常见或更少见。若把这些 4 调得更大，先验才会开始表达“预期它们实际相等”。

现在怎样表达数据的概率，也就是 likelihood？对于每个观测选择 yi，每种策略 s 都意味着看到 yi 的某个概率。把它记作 Pr(yi | s)，即在假定某个具体策略 s 时数据的概率。例如，若假定 s = 1，也就是“跟随多数”，那么 Pr(yi = 2 | s = 1) = 1。这只是用数学说：采用“跟随多数”策略的儿童，总会选择多数颜色。

可是我们不知道 s，也不能直接观测它。不过，模型确实给出了每种 s 的概率，也就是单纯形 p 的各个元素。因此，要得到数据的无条件概率 Pr(yi)，只需用 p 对未知策略 s 取平均：

Pr(yi) = Σs=15 ps Pr(yi | s)

这句话读作：“yi 的概率，是在每种策略 s 下 yi 的条件概率的加权平均。”这个表达式与前面章节一样，是一个混合。有时你会看到“把未知策略取边际”这种说法；它只是指对各策略取平均，并用每种策略的某个概率作为权重。上式中，p 的各个值正提供这些权重。

至此统计模型已经齐备。把它写成更常见的形式：

yi ∼ Categorical(θ)
θj = Σs=15 ps Pr(j | s)　　对于 j = 1…3
p ∼ Dirichlet([4, 4, 4, 4, 4])

向量 θ 保存以 p 为条件时每种行为的平均概率。作为生成模型，上式意味着所有儿童都完全相同：每名儿童在每轮试验中都以概率 ps 使用策略 s。儿童之间当然存在个体差异；但这些数据没有每名儿童的重复观测，所以我们很难做得更好。如果真有重复观测，就可以给每名儿童分配一个独有的单纯形 p，启动部分汇聚，然后欣赏烟花。

过度思考：Boxes 模型的 Stan 代码

Stan 模型需要三个代码“块”。依次来看。第一块是数据块，只为观测变量命名并声明类型。本模型如下：

data{
    int N;
    int y[N];
    int majority_first[N];
}

整数 N 只是观测案例数，即 data(Boxes) 的行数。随后把结果 y 与预测变量 majority_first 声明为长度 N 的整数向量。也可以把长度硬编码为 629，但那样案例数每次变化，都得修改模型代码。

Stan 模型所需的第二块是参数块。它类似数据块，但声明未观测变量，也就是我们要取得后验样本的变量。本模型只有单纯形 p：

parameters{
    simplex[5] p;
}

第三块是核心——模型块。它计算观测变量（数据）与未观测变量（参数）的对数概率，也就是 Bayes 定理的分子；Stan 用它运行 Hamiltonian 模拟（见第 9 章）。下面分段来看。首先，在顶部声明一个向量，保存每种策略的概率计算；处理数据的每一行时都会复用它，算出不同概率：

model{
    vector[5] phi;

然后指定先验：

    // prior
    p ~ dirichlet( rep_vector(4,5) );

接下来进入核心。循环遍历每一行；对于第 i 行，计算观测值 y[i] 的对数概率。每种策略都有自己的 if...then，用于指定以该策略为条件时数据的概率：

    // probability of data
    for ( i in 1:N ) {
        if ( y[i]==2 ) phi[1]=1; else phi[1]=0; // majority
        if ( y[i]==3 ) phi[2]=1; else phi[2]=0; // minority
        if ( y[i]==1 ) phi[3]=1; else phi[3]=0; // maverick
        phi[4]=1.0/3.0;                         // random
        if ( majority_first[i]==1 )             // follow first
            if ( y[i]==2 ) phi[5]=1; else phi[5]=0;
        else
            if ( y[i]==3 ) phi[5]=1; else phi[5]=0;

现在要纳入参数 p。把每个 log(ps) 加到上面算出的对数概率中，再把平均概率加到 target；target 只是 Stan 对“总对数概率”的称呼：

        // compute log( p_s * Pr(y_i|s) )
        for ( s in 1:5 ) phi[s] = log(p[s]) + log(phi[s]);
        // compute average log-probability of y_i
        target += log_sum_exp( phi );

log_sum_exp 按正文定义，计算数据的边际对数概率 log Pr(yi)。它接收向量 phi——其中保存每种策略各自的对数概率——并返回这些概率在概率尺度上求和后再取对数的结果。在这种带离散参数的 Stan 模型中，它十分常见。

要加入预测变量，选择很多，因此再次依靠真实理论有助于聚焦。最简单的修改，是让单纯形 p 随性别之类的离散类别变化。此时在数据块中加入变量 gender，并在参数块中给 p 增加一个维度：

    simplex[5] p[2];

然后在模型块中，用 p[gender[i],s] 同时按策略和性别索引 p：

    for ( s in 1:5 ) phi[s] = log(p[gender[i],s]) + log(phi[s]);

这个模型作为 data(Boxes_model_gender) 收录在 rethinking 包中。年龄这样的连续协变量会带来更多选择：Gaussian process、样条、多项式都能处理，却各要用不同方式编码。Stan 模型 data(Boxes_model_age) 展示了一个简单的线性年龄趋势：先给每个 p 在 logit 尺度上指定线性模型，再用多元逆 logit 变换到单纯形尺度。这完全是地心式做法；如果有更强的理论，它会有所帮助。

16.2.4 状态空间模型

上面的 Boxes 模型类似于更宽泛的一类模型，称为状态空间模型。这类模型假设有多个隐藏状态产生观测。状态通常是动态的，会随时间改变。当状态是离散类别时，模型可称为隐 Markov 模型（HMM）。许多时间序列模型都是状态空间模型，因为真正的时间序列状态并不可观测，能看到的只有带噪声的测量。本章稍后会给出一个例子。
