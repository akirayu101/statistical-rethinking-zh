第六章

闹鬼的 DAG 与因果恐怖

当前译稿对应原书 PDF 第 181–207 页

最有新闻价值的科学研究，似乎也最不可信。如果一项研究为真时越可能要你的命，它反而越不可能为真；主题越无聊，结果似乎越严谨。这种广为相信的负相关怎么会存在？人们关心的主题并没有理由产生更不可靠的结果。也许热门主题吸引了更多、更糟糕的研究者，就像苍蝇被蜂蜜气味吸引一样？

其实，只要同行评审者同时在意新闻价值与可信度，这种负相关就会出现。无论基金评审还是期刊评审，只要编辑与审稿人同时看重二者，筛选行为本身就足以让最有新闻价值的研究显得最不可信。事实上，很难想象科学同行评审怎样才能完全避免制造这种负相关。亲爱的读者，这个事实也会帮助我们理解多元回归的危险。

下面用一个简单模拟说明。85 假设基金评审小组收到 200 份科研提案。所有提案中，可信度（严谨性、学术水平、成功的可信程度）与新闻价值（社会福利价值、公众兴趣）完全不相关。小组给二者相同权重，按合计得分排序，并资助最高的 10%。

本节末尾给出模拟代码。图 6.1显示全部提案，蓝色表示获选者，并为获选提案画出简单线性回归线。负相关出现了——本图约为 −0.78。强筛选会在筛选标准之间诱发负相关。为什么？若跨过门槛的唯一办法是高分，那么只在一项上得高分，会比两项都得高分更常见。因此，在获资助提案中，新闻价值最高的研究完全可能拥有低于平均的可信度；同样，最可信的研究也可能比平均水平更缺乏新闻价值。

这一普遍现象早已为人所知，有时称为 Berkson 悖论。86 不过，把它叫作选择扭曲效应更容易记忆。一旦理解它，就会到处看到它：为什么好地段的许多餐厅食物很差？食物不够好的餐厅只有位于好地段才能生存；食物极佳的餐厅则能在坏地段活下来。选择扭曲毁掉了你的城市。

这与多元回归有什么关系？不幸的是，关系无处不在。上一章展示了多元回归的一些惊人能力：它能揭穿虚假相关，也能澄清掩盖效应。这很容易诱使人相信，只要有疑问，就把一切都放进模型，让回归神谕自行整理。

回归不会替你整理。回归确实是神谕，却是残酷的神谕；它说谜语，并乐于惩罚提出坏问题的人。选择扭曲效应可以发生在多元回归内部，因为加入预测变量的行为会在模型中诱发统计筛选，这种现象有个并不友好的名字：碰撞点偏差。它可能误导我们，让我们相信新闻价值与可信度在总体上负相关，而实际上这只是以某个变量为条件的结果。这个事实既极其令人困惑，又是负责任地使用回归所必须理解的。

本章与下一章都讨论：在没有清晰因果模型时，仅仅向回归加入变量会发生哪些糟糕事情。本章考察三种危险：多重共线性、处理后偏差与碰撞点偏差。最后会把这些例子纳入一个框架，判断为了有效推断，哪些变量必须加入模型，哪些绝不能加入。但这个框架不会替我们完成最重要的一步：它不会凭空给出有效模型。

6.1 多重共线性

回归模型通常有许多潜在预测变量可供加入。例如，灵长类乳汁数据中有 7 个变量可以预测我们选作结果的任何一列。为什么不直接拟合包含全部 7 个变量的模型？因为存在若干危险。这里首先关注多重共线性：两个或更多预测变量之间存在非常强的相关。其后果是，后验分布似乎会表明没有任何变量可靠地与结果相关，即使现实中所有变量都与结果强相关。

这种令人沮丧的现象源于多元回归工作的细节。事实上，多重共线性本身没有“错误”；模型用于预测时仍能正常工作，只是会让解释变得令人抓狂。理解多重共线性之后，也会更深入地理解一般回归模型。先从简单模拟开始，再回到灵长类乳汁数据观察真实数据中的多重共线性。

这个怪问题的答案同样怪，却完全合乎逻辑。后验会考虑参数的每一种可能组合，并在给定模型与数据后为每种组合分配相对可信程度。查看 bl 与 br 的二元后验会有所帮助：

图 6.2 左侧显示，两个参数的后验高度相关，所有可信的 bl 与 br 值都沿着一条狭长脊分布：bl 大时，br 必须小。因为两条腿几乎包含完全相同的信息，坚持把二者都放入模型，就会有近乎无穷多组 bl 与 br 产生相同预测。

可以把这种现象理解为近似了下面的模型：

yi ∼ Normal(μi, σ)
μi = α + β1xi + β2xi

y 是身高之类的结果，x 是腿长之类的单一预测变量。这里把 x 用了两次，是两条近乎相同的腿造成问题的完美例子。从计算机角度看，这个模型只是：

yi ∼ Normal(μi, σ)
μi = α + (β1 + β2)xi

这里只是从两项中提出了 xi。β1 与 β2 无法分开，因为它们从不单独影响均值；只有二者之和影响 μ。因此后验会报告大量组合，只要它们的和接近 x 与 y 的实际关联即可。

科学文献中能见到各种应对多重共线性的可疑方法，很少采取因果视角。有些领域教学生在拟合模型前检查成对相关，删除高度相关的预测变量。这是错误的。成对相关本身不是问题；真正重要的是条件关联，而不是相关。即便如此，正确做法仍取决于共线性由什么造成。仅凭数据内部的关联不足以决定怎样处理。

乳汁例子中，可能存在哺乳动物母亲必须遵守的乳汁组成核心权衡。某物种若频繁哺乳，乳汁往往更稀、能量更低，并富含糖（乳糖）；若哺乳次数少而每次时间短，乳汁就需要更高能量，因此脂肪很高。这意味着类似下面的因果模型：

多重共线性属于一族模型拟合问题，这族问题常称为不可识别性。参数不可识别，意味着数据与模型的结构无法估计参数值。有时这是模型编码错误造成的，但许多重要模型即使编码完全正确，也包含不可识别或弱可识别参数。即使模型正确，自然也不欠我们容易的推断。

一般而言，现有数据不保证包含很多关于目标参数的信息。若信息很少，贝叶斯机器会返回与先验非常相似的后验。因此比较后验与先验是个好主意，可以看出模型从数据中提取了多少信息。二者相似并不表示计算错误——你得到了所提问题的正确答案；但它可能促使你提出更好的问题。

6.2 处理后偏差

人们通常会担心遗漏预测变量造成的错误推断，这类错误常称为遗漏变量偏差，上一章已有示例。相比之下，人们较少担心把其他变量的后果纳入模型所造成的错误推断。这里称之为处理后偏差。87 所有类型的研究都必须警惕它。精心控制的实验和无控制的观察研究一样容易被毁掉；无论怎样收集数据，把变量盲目扔进因果沙拉都不是好主意。

“处理后”这个说法来自实验设计。设想在温室里种植物，想知道不同抗真菌土壤处理对生长的影响，因为真菌会降低生长。植物播种发芽后先测初始高度，再施加不同土壤处理，最后测量高度与是否出现真菌。四个变量是初始高度、最终高度、处理和真菌存在。结果是最终高度。若目标是对处理做因果推断，就不应纳入真菌，因为它是处理后的效应。

6.2.1 先验诞生

设计模型时，最好假装并不知道上面的真实数据生成过程。真实研究中也不会知道它，但会有大量科学信息指导建模。因此认真对待这次模拟分析。

无论测量尺度是什么，时间 t = 1 时植物都应比 t = 0 更高。若把参数放在相对于初始高度的比例尺度，而非数据的绝对尺度上，先验会更容易指定。暂时忽略预测变量，只关注高度，可以建立：

h1,i ∼ Normal(μi, σ)
μi = h0,i × p

p = h1,i/h0,i 测量最终高度相对初始高度的比例。p = 1 表示没有变化，p = 2 表示高度翻倍。以 1 为先验中心意味着预期不生长，这比已有知识更保守；同时应允许 p < 1，以防实验灾难性地杀死植物，并保证 p > 0。对数正态分布始终为正，因此令 p ∼ Log-Normal(0, 0.25)：

这个先验容许从缩短约 40% 到增长约 50%。拟合模型，看看它怎样测量实验中的平均生长：

平均增长约 40%。接下来加入处理与真菌变量，试图同时测量处理与真菌本身的影响；这些参数也放在比例尺度上，表示比例生长的变化。于是 p 的模型变为：

h1,i ∼ Normal(μi, σ)
μi = h0,i × p
p = α + βTTi + βFFi
α ∼ Log-Normal(0, 0.25)
βT ∼ Normal(0, 0.5)
βF ∼ Normal(0, 0.5)
σ ∼ Exponential(1)

生长比例 p 现在是预测变量的函数，看起来与其他线性模型相同。斜率先验几乎肯定过于平坦：95% 的先验质量落在 −1（减少 100%）到 +1（增加 100%）之间，三分之二落在 −0.5 到 +0.5 之间。读完本节后，不妨回头从这些先验做模拟。下面近似后验：

6.2.2 被后果阻断

问题在于，真菌主要是处理的后果，也就是说，真菌是处理后变量。因此，控制真菌时，模型暗中回答的是：“既然已经知道植物是否长了真菌，土壤处理还重要吗？”答案是“不重要”，因为土壤处理正是通过减少真菌来影响生长。但依据实验设计，我们真正想知道的是处理对生长的影响。要正确测量它，应省略处理后变量真菌。推断结果如下：

6.2.3 真菌与 d-分离

用 DAG 来看这个问题会更清楚。下面也展示如何用 R 包 dagitty 绘图，因为接下来要用它做图分析：

把处理后效应 F 纳入模型，就会阻断处理到结果的路径。换成 DAG 的说法：一旦知道真菌状态，再知道处理与否，就不能提供任何关于结果的新信息。

更具 DAG 味道的说法是，以 F 为条件会产生 d-分离。“d”代表 directional（有向）。88 d-分离表示有向图中的某些变量与另一些变量独立，它们之间不存在开放路径。本例中，只有在以 F 为条件时，H1 才与 T d-分离。以 F 为条件有效阻断了有向路径 T → F → H1，使 T 与 H1 独立，即 H1 ⫫ T | F。原因是 T 中没有任何关于 H1、却不同时存在于 F 中的信息。可以查询这个 DAG 隐含的条件独立性：

共有三条，第三条是当前讨论的重点；另外两条则提供了检验 DAG 的方法。F ⫫ H0 和 H0 ⫫ T 表示：只要不以任何变量为条件，初始高度 H0 就不应与处理 T 或真菌 F 相关。

处理后变量的问题显然同样适用于观察研究和实验。实验中通常较容易区分初始高度这类处理前变量与真菌这类处理后变量；观察研究中则更难判断。不过，实验本身也有一些隐蔽陷阱。

再思考：模型选择也无济于事

下一章会介绍用信息准则进行模型选择。与其他模型比较和选择方案一样，这些准则有助于对照和选择模型结构；但它们救不了刚才的例子，因为包含真菌的模型不仅对样本拟合更好，还会做出更好的样本外预测。模型 m6.7 之所以误导，是因为它问错了问题，而不是因为预测很差。正如第 1 章所论证的，预测与因果推断根本不是同一项任务。任何统计程序都无法替代科学知识以及对它的认真运用。我们需要多个模型，是因为它们帮助理解因果路径，而不只是为了挑一个模型做预测。

6.3 碰撞点偏差

本章开头说过：只要资助评审和期刊评审这些筛选过程同时看重可信度与新闻价值，科学研究就会在二者之间表现出负关联。现在要解释，同样的筛选现象如何发生在统计模型内部。它一旦发生，就会严重扭曲推断，这种现象称为碰撞点偏差。

为这个例子画一张 DAG。假设提交给资助评审委员会的全部研究计划中，可信度 T 与新闻价值 N 在统计上独立；二者都会影响计划是否获资助，即筛选 S：

两支箭头都射入 S，说明它是碰撞点。核心概念不难：以碰撞点为条件，会在它的各个原因之间制造统计关联，但不一定制造因果关联。本例中，一旦知道某项计划已获选 S，再知道它的可信度 T，也就能提供新闻价值 N 的信息。为什么？例如，获选计划若可信度低，新闻价值就必定高，否则不会得到资助。反过来也一样：若新闻价值低，我们会推断它的可信度必定高于平均水平，否则不可能获选。

正是这种信息现象，在获选计划中产生了 T 与 N 的负关联。因此必须留意筛选观测样本的过程，因为它们可能扭曲变量之间的关联。同样的现象还会在把碰撞点作为预测变量纳入统计模型时，制造误导性关联；稍不留神，就会得出错误的因果推断。下面看一个扩展示例。

6.3.1 虚假悲伤的碰撞点

考虑衰老如何影响幸福感。如果有一项大型调查，让人们给自己的幸福程度评分，年龄会与幸福感相关吗？如果相关，这种关联是因果的吗？这里要展示：控制一个看似合理的幸福感混杂因素，实际上反而会使年龄影响的推断产生偏差。89

祖父母例子也是 Simpson 悖论：加入另一个预测变量（这里是 P）会逆转某个预测变量（G）与结果（C）之间的关联方向。Simpson 悖论通常以新增预测变量有帮助的情形呈现，但本例中它反而误导。Simpson 悖论是一种统计现象；要判断关联逆转是否正确反映因果，仅凭统计模型还不够。91

6.4 直面混杂

本章与上一章已经有多个例子，展示如何用多元回归处理混杂；但也看到多元回归如何制造混杂——控制错误的变量会毁掉推断。希望这些例子已经吓得你不敢再把所有变量都塞进模型，指望回归自己理清一切；同时也让你相信，只要足够谨慎、知识足够充分，有效推断仍然可能。

但为什么有时遗漏变量、有时加入变量，却会产生同一种现象？还有其他因果怪物潜伏在图中吗？我们需要一套一致原则。

把混杂定义为：结果 Y 与目标预测变量 X 之间的关联，不同于通过实验决定 X 取值时会观察到的关联。92 例如，设我们关心教育 E 与工资 W 的关联。典型总体中有许多未观测变量 U 同时影响二者，例如居住地、父母和朋友。DAG 如下：

若让 W 对 E 回归，因果效应估计会被 U 混杂。因为连接 E 与 W 的路径有两条：（1）E → W；（2）E ← U → W。这里“路径”只是从一个变量走到另一个变量所经过的任意变量序列，忽略箭头方向。两条路径都会产生统计关联，但只有第一条是因果路径。为什么第二条不是？若只存在第二条，改变 E 并不会改变 W；教育对工资的任何因果影响都只沿第一条路径运行。

怎样隔离因果路径？最著名的办法是做实验。若能随机分配教育水平，图就变为：

操纵会移除 U 对 E 的影响：当我们自己决定教育水平时，未观测变量不再影响教育。移除 U → E 也就移除了路径 E ← U → W，阻断第二条路径。路径被阻断后，E 与 W 之间只剩一种信息流动方式，二者的关联便能有用地度量因果影响。操纵通过阻断另一条路径消除了混杂。

幸运的是，不必真的操纵 E，统计方法也能达到同样结果。最明显的做法是把 U 加入模型，以 U 为条件。这同样会阻断信息经 U 在 E 与 W 之间流动，阻断第二条路径。

要理解为什么以 U 为条件会阻断 E ← U → W，可把这条路径单独看作完整模型。一旦知道 U，再知道 E 就不会提供关于 W 的额外信息。例如让 U 表示地区平均财富：富裕地区学校更好，教育程度 E 更高；工作薪酬也更高，工资 W 更高。不知道某人居住地区时，教育会提供工资信息，因为二者跨地区相关；但知道居住地区后，假设二者之间没有其他路径，再知道教育就不再提供工资信息。这正是以 U 为条件“阻断”路径的含义：给定 U 后，E 与 W 独立。

6.4.1 关闭后门

阻断目标预测变量 X 与结果 Y 之间的所有混杂路径，称为关闭后门。这个比喻是说，我们不希望虚假相关从非因果路径偷偷溜进来；这种路径会从预测变量 X 的“背面”进入。上例中的 E ← U → W 就是后门路径，因为箭头射入 E，同时又把 E 与 W 连接起来。这条路径会造成混杂。

好消息是：只要给定因果 DAG，就总能判断为了关闭所有后门路径需要控制哪些变量（如果需要的话），也能判断为了让目标路径保持开放，绝对不能控制哪些变量。

还有一个好消息：所有可能的路径，都只由四类变量关系组合而成。只需理解四件事，以及信息怎样在其中流动。图 6.6 展示了每一种基本关系；任何 DAG 无论多大、多复杂，都由它们构成。按从左到右逐一看：

  1. 分叉：X ← Z → Y。这是经典混杂：共同原因 Z 使 X 与 Y 相关。以 Z 为条件后，知道 X 不再提供 Y 的信息；给定 Z，二者独立。
  2. 管道：X → Z → Y。植物生长和处理后偏差就是如此：处理 X 影响真菌 Z，真菌再影响生长 Y。以 Z 为条件同样阻断 X 到 Y 的路径。因此，无论分叉还是管道，以中间变量为条件都会阻断路径。
  3. 碰撞点：X → Z ← Y。与前两类不同，不以 Z 为条件时，X 与 Y 没有关联；以碰撞点 Z 为条件会打开路径，让信息在 X 与 Y 之间流动。
  4. 后代变量：以后代变量为条件，类似于以变量本身为条件，只是作用较弱。后代是受另一个变量影响的变量。在图 6.6 最右侧，以 D 为条件也会在较弱程度上控制 Z，因为 D 包含一些关于 Z 的信息。这会部分打开 X 到 Y 的路径，因为 Z 是碰撞点。对非碰撞点也一样：若以管道中 Z 的后代为条件，也会像较弱地关闭管道。

无论因果 DAG 看起来多复杂，都由这四类关系构成。既然知道每一类怎样打开和关闭，你（或计算机）就能判断为了关闭后门应该控制哪些变量、又不该控制哪些变量。步骤如下：

  1. 列出连接 X（目标潜在原因）与 Y（结果）的全部路径。
  2. 判断每条路径是开放还是关闭；除非包含碰撞点，否则路径开放。
  3. 判断每条路径是否为后门路径；后门路径有箭头射入 X。
  4. 若存在同时开放的后门路径，决定以哪些变量为条件将其关闭。

6.4.2 两条路

下面的 DAG 包含目标暴露 X、目标结果 Y、一个未观测变量 U，以及三个已观测协变量 A、B、C。

我们关心的是因果效应 X → Y。为了正确推断它，需要把哪些已观测协变量加入模型？除直接路径外，从 X 到 Y 还有两条路径：

  1. X ← U ← A → C → Y
  2. X ← U → B ← C → Y

两条都是可能混杂推断的后门路径。接着判断哪条开放：开放的后门必须关闭，已经关闭的后门则绝不能意外打开而制造混杂。

第一条经过 A，其中没有碰撞点，因而开放。顶部是一个分叉，两侧各有一条管道；信息会沿路径流动并混杂 X → Y。要关闭它，需以路径上的某个变量为条件。U 未观测，不能控制，只剩 A 或 C，任一个都能关闭后门。R 包 dagitty 的 adjustmentSets 可以分析图并找出需要控制的变量：

以 C 或 A 为条件都足够。从效率角度看，控制 C 更好，因为它还可能提高 X → Y 估计的精度。控制 U 也能奏效；但代码已告诉 dagitty 它未观测，所以调整集中没有建议 U。

再看经过 B 的第二条路径，其中包含碰撞点 U → B ← C，所以已经关闭。它不是需要关闭的开放后门，因此 adjustmentSets 没有提到 B。无需控制 B；若真的控制，并非无害，而是会打开路径，制造混杂。此后对 X → Y 的推断会改变，但没有 DAG 就不知道这种改变是在帮助还是误导。加入一个变量使 X → Y 的系数改变，并不总意味着系数更好了——你可能只是以碰撞点为条件。
